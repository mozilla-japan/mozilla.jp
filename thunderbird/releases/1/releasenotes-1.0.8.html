Thunderbird リリースノート

バージョン 1.0.8

Thunderbird はフル装備の E-メール / RSS / ニュースグループクライアントで、かつてないほど安全で素早く、そして簡単にメールを扱うことができます。Thunderbird についての より詳しい情報はこちら を参照してください。

このリリースノートには、Thunderbird 1.0.8 に関する 最新情報、ダウンロード と インストール方法、既知の問題、ユーザーサポートとフィードバック などの情報が載っています。どんなバグでも Bugzilla にバグを報告する前にこのリリースノートと バグの登録方法を読んでください。

Thunderbird に関するフィードバックを歓迎します。MozillaZine が快くホストしてくれている Mozilla Thunderbird サポートフォーラム に参加してみてください。

1.0.8 の新機能と改良点

Thunderbird 1.0.8 は、ユーザーの皆さんにより安全なインターネット体験をお届けする私たちの取り組みの一環として提供される、安定性とセキュリティを向上させたアップデート版です。すべてのユーザーにこの最新版へのアップグレードを推奨します。

  • 安定性の向上
  • いくつかのセキュリティ問題の修正

注目すべきバグの修正のより詳細なリスト が The Rumbling Edge にあります。

ダウンロードとインストール

システム要件

インストールする前にあなたのコンピューターが システム要件 を満たしているか確認してください。

ダウンロード

Mozilla Foundation では Windows、Linux、Mac OS X 用の Thunderbird を提供しています。

英語版
Windows: インストーラ
Linux (GTK2+XFT): tar.gz 形式
Mac OS X: 圧縮ディスクイメージ
日本語版
(準備中です)

インストール

Thunderbird のインストーラあるいは圧縮アーカイブをダウンロードしたら次の説明に従ってインストールしてください。

注意：Mozilla Thunderbird は、プロファイル情報がなければ Thunderbird 専用のユーザープロファイル情報を作成します。これまでに Mozilla Suite をインストールしてあった場合でも、Thunderbird は Mozilla Suite のプロファイルとは干渉しません。

Windows

インストーラ Thunderbird Setup 1.0.8.exe をダブルクリックしてインストールを始めます。

Mac OS X

Thunderbird の圧縮ディスクイメージをダブルクリックしてマウントします。すでにブラウザーがこのディスクイメージを展開、マウントしているかもしれません。Thunderbird 1.0.8 ディスクイメージをダブルクリックして Thunderbird アプリケーションをハードディスクにドラッグします。ディスクイメージ内のアイコンをダブルクリックしてはいけません。Firefox アプリケーションは、起動する前に必ず、ディスクイメージからハードディスクにドラッグしてください。Dock に追加したい場合は、Thunderbird のアイコンを Dock にドラッグしてください。

その他のプラットフォーム

圧縮されたアーカイブを展開し thunderbird を実行します。

アンインストール

Thunderbird をアンインストールするには次の手順に従ってください：

Windows

スタート メニューから コントロールパネル を選択します。コントロールパネルが表示されたら プログラムの追加と削除 をダブルクリックします。 リストの中の "Mozilla Thunderbird (1.0.8)" を見つけて 削除 をクリックしてアンインストールします。

Mac OS X

Thunderbird アプリケーションをゴミ箱にドラッグします。

その他のプラットフォーム

thunderbird ファルダを削除します。

これらの手順でアンインストールを完了しても、また次に Thunderbird をインストールする場合を考えて、あなたのプロファイルは削除されず、そのまま保存されます。Thunderbird のプロファイルを削除したい場合は、以下の「プロファイルフォルダー」の項目に書かれているフォルダーを削除してください。なお、一度プロファイルを削除してしまうと、ブックマークや保存されたパスワード、個人設定、その他のデータがすべで消去されてしまいますので、次に Thunderbird を使う場合は新しいプロファイルで起動することになります。

拡張機能とテーマ

Thunderbird の新バージョンにアップグレードする際、お使いの拡張機能やテーマは、Thunderbird の新バージョンと互換性があること、または、互換性のある新しい拡張機能やテーマが提供されていることが確認されるまで、すべて無効化されます。これは、Thunderbird がバージョンごとに変更点を含んでおり、その変更された部分の機能を統合した一部の拡張機能やテーマは、Thunderbird の新バージョンで使うと問題を引き起こす可能性があるためです。お気に入りの拡張機能やテーマがアップデートされておらず、Thunderbird の新バージョンと互換性がない場合は、作者に連絡を取ってアップデートするよう促してください。

既知の問題

ここには、Thunderbird 1.0.8 に関する既知の問題の一部が載っています。新たにバグを報告する前にこのリストをご覧になり、今後も定期的にチェックするようにしてください (リリースに新しいバグが見つかった場合は更新されます)。

【訳注: Thunderbird 1.0.8 に含まれる日本語環境特有の問題をまとめた トラブルシューティング を公開しています。こちらも併せてご覧ください】

すべての OS

  • マルチユーザーシステムで、Thunderbird をアクセス制限のある場所にインストールした場合、その場所にアクセス可能なユーザーがまず最初に Thunderbird を起動する必要があります。そうすることで、初回起動ファイルがすべて生成されます。この作業を行わないと、インストールされた場所に書き込み権限を持たないユーザーが Thunderbird を起動しようとした場合、Thunderbird が必要とする初回起動ファイルが正しく生成されません。

  • アップグレードの際、拡張機能やテーマはすべて無効化されます。これは問題ではありませんが、見た目にはそのように受け取られる場合もあります (そのため、ここに書かれているのですが)。詳しくは上の「拡張機能とテーマ」をご覧ください。

  • デフォルトのスペルチェック辞書は英語のみとなっています。英語以外の辞書を追加するには インストール方法 を参照してください。

  • 共通受信トレイと POP3 アカウント用受信トレイの使用を切り替える場合は、必ずメッセージを他のフォルダーにコピーしておいてください。

  • Thunderbird は、ユーザーのプライバシーを守るため、デフォルトではすべてのメッセージの Cookie をブロックするようになっています。この設定は、Cookie を必要とする一部の RSS フィードで問題となる場合があります。次の行をプロファイルの prefs.js ファイルに追加することで、Cookie を有効にできます: user_pref("network.cookie.cookieBehavior", 3);

  • 受信通知機能 (MDN) は POP3 アカウントでは正しく機能しません。

Windows

  • 共有マシンで、アクセス権の限られたユーザーが自分の書き込み権限のある場所に Thunderbird をインストールした場合でも、マイナスの副作用が発生する場合があります (既定のメールクライアントその他のキーが正しく設定されないなど)。それでも Thunderbird は使えます。アクセス権の限られたユーザーがインストールする場合、アクセス制限のある場所や共有の場所にインストールしてはいけません。インストールに失敗する可能性があります。

  • Windows ユーザーは、Thunderbird 用の Palm Sync 拡張機能 をインストールすることによって、Palm デバイスとアドレス帳を同期させることができます。このファイルをローカルに保存し、Thunderbird の拡張機能ダイアログを使用してインストールしてください。この拡張機能は拡張機能の一覧には表示されません。実際に機能を有効にするには、インストール後に一度 Thunderbird を再起動する必要があります。

Mac OS X

  • Thunderbird をディスクイメージから実行しないでください。万が一実行してしまうと、無限再起動ループが起こってしまいます (Dock 内で Thunderbird のアイコンが跳ねたり消えたりを繰り返す現象が見られます)。この無限ループを止めるには、ターミナルを開いて "killall thunderbird-bin" と入力してください。それから再度、書き込み権限のある場所に Thunderbird をインストールしてください。アクセス制限のあるマルチユーザーシステムにインストールする場合は、管理者として共有の場所にインストールしてください。そうすれば、すべてのユーザーがアクセス可能になります。

Linux/Unix システム

  • GNOME との統合は Fedora Core 3 では正しく機能しません。Fedora Core 3 ユーザーは、linc-1.0.3-3.1.i386.rpm をダウンロード・インストールする必要があります。この RPM をインストールしたら、Thunderbird をインストールしたディレクトリーで次のコマンドを実行してください (書き込み権限が必要です)。

    touch .autoreg

    次に Thunderbird を起動したときから GNOME 統合が有効になります。

よくある質問

どのような協力ができますか?
私たちは少しでも多くの配布経路を必要としています。できる限り多くの友人や家族、同僚を Thunderbird に乗り換えさせることがあなたの使命です。もしあなたが学生なら、学校で Thunderbird を配布してください。リリースについて Slashdot [日本語版] やその他のニュースサイトに記事を投稿してください。あなたのブログから情報を流してください。例えばあの Kazaa のように、インターネットを通じた大規模な配布が可能です。
なぜ私が送ったメールに返事をくれないのですか?
フォーラム をご利用ください。Thunderbird チームは常時このフォーラムをチェックしています。私たちは毎日多くのメールを受け取っているので、あなたが送ったメールは見過ごされる可能性もあります。
テーマとアドオン (拡張機能) はどこで手に入りますか?
テーマと拡張機能は Mozilla Update からダウンロードできます。
誰が Thunderbird の作業をしているのですか?
非常に多くの人が関わっています。Thunderbird に貢献している人たちの一覧を見るには、[ヘルプ] メニューの [Mozilla Thunderbird について] に書かれているクレジットをご覧ください。
