Thunderbird リリースノート

バージョン 0.6

Thunderbird はフル装備の E-メール / ニュースグループクライアントで、かつてないほど安全で素早く、そして簡単にメールを扱うことができます。Thunderbird についてのより詳しい情報はこちらを参照してください。

Mozilla の基礎的なコードが成熟しているおかげで Thunderbird は十分使用できる状態にあります。しかしながら、このリリースはまだテクノロジー・プレビューであり、従って欠陥があることも十分考えられ ます。詳しい情報は License Agreement を参照してください。

最新情報

このリリースのハイライトは以下のとおりです：
  • Windows インストーラー

    Thunderbird に Windows 用のインストーラーができました。これでもっと簡単に Thunderbird を使いはじめることができます！

  • MacOS X 用の新しいデフォルトテーマ

    Mac OS X のルックスによく似合う新しい Pinestripe テーマを採用しました。

  • 改良されたジャンクメールコントロール

    学習機能つきジャンクメールコントロールのアルゴリズムが大きく再設計され、ジャンクコントロールがより早く学習するように、そしてより多 くのスパ ムをとらえるようになりました。

  • 新しいブランド・アイデンティティ

    ブランド・アイデンティティに対する Mozilla Foudation の目標にあわせて、Thunderbird に新しいロゴとアートワークが備わりました。素晴らしい仕事をしてくれた Mozilla Visual Identity チームに感謝します。

  • その他の新機能...

    IMAP ユーザーは IMAP IDLE コマンドのサポートにより恩恵を受けられるようになりました。これによってメールサーバーは、新規メールの到着などの通知を出すことができるようになりま す。

    Thunderbird は、サーバーにあるすべてのニュースグループに適用するサーバー-ワイド ニュースフィルターをサポートしています。

    Thunderbird は、IMAP や POP3、そして SMTP 用の新しいクロスプラットフォーム NTLM 認証メカニズムを使用したセキュア・パスワード認証を備えています。

    メールフィルターがメッセージにジャンクマークを付けられるようになりました。

    オフラインサポートは Windows インストーラのオプションでダウンロードするコンポーネントとなり、機能拡張ではなくなりました。

    Mac OS X でシステム Dock 上に新規メールが通知されるようになりました。

    DOM インスペクターはテーマ作者のために Windows インストーラのオプションでダウンロードするコンポーネントとなりました。

    ツール(Tools) > オプション(Options) > メッセージの作成(Compose) > HTML オプション(HTML Options) を選択すると、フォントやサイズ、色など、HTML でのメール作成時のデフォルトオプションを設定することができます。

    メール作成ウインドウから添付書類を直接開けるようになり、送信する前にその内容を確認することができます。

    アカウント毎の複数 ID をサポートするようになりました。これにより、最終的に同じアカウントに入る複数のメールアドレスを簡単に持つことができます。設定の仕方についてはこちらを参照してください。

  • 最近フィックスされたバグ

    メッセージをオンライン送信済みフォルダーへコピーできなかった場合、もう一度コピーしてみるかどうか Thunderbird が尋ねるようになりました。

    Windows 用の 0.6 リリースでは Simple MAPI に対するいくつもの改良が行われ、Microsoft Office の古いバージョンと連携できるようになりました。

    スプレッドシートからのデータを HTML でのメール作成ウインドウにペーストしても、実際のスプレッドシートデータの前に乱雑なゴミが ペーストされなくなりました。

    LDAP 自動コンプリート使用時や LDAP ディレクトリー上での検索時に LDAP 接続が開かれたままになるいくつかの問題を直しました。

    ソース閲覧の動作を改善しました。

    メールフィルターによって消去もしくは既読マークが付けられた POP3 メッセージに対するメール通知は行われなくなりました。

    Linux GTK2 で "Mark All Read (すべて既読にする) " のキーボードショートカットが使えるようになりました。

システム動作環境

インストールする前にあなたのコンピューターがシステム 動作環境を満たしているか確認してください。

ダウンロード、インストールとアンインストール

アップグレードする場合： 前のバージョンのプ ログラムファイルを含むディレクトリーに Mozilla Thunderbird をインストールしないでください。前のバージョンに上書きすると問題を引き起こす可能性があります。以前にインストールしたディレクトリーを再利 用するには、 そのディレクトリーを削除して再度作り直すか、空にするか、移動するか、あるいは違う名前を付けます。この手順を無視した場合には絶対に Bugzilla にバグを登録しないでください。

このプログラム自体のディレクトリーにはプロファイル情報は含まれていません。従って新たにインストールしても、今あるアカウント、アカウントの設定、オプ ション、メール、そしてニュースメッセージが損なわれることはありません。このリリースではプロファイルを変更しなくても正しく作動します。

重要： Thunderbird の前のバージョンを使用していて、テーマもしくは機能拡張をインストールしている場合は，以下の手順に従ってください。さもないと Thunderbird は正しく動作しないかもしれません。まず プロファイル の場所 (和訳) を 見つけてください。"chrome" と名付けられたサブディレクトリーがあるはずです。そして "chrome" 内のすべてを削除します。この操作がメールのデータや設定に影響を与えることはありません。

Thunderbird のインストール

Thunderbird のインストーラあるいは圧縮アーカイブをダウンロードしたら次の説明に従ってインストールしてください：

注意： Mozilla Thunderbird は専用のユーザープロファイル情報を作成します。もし無くても Mozilla アプリ - スイートのプロファイルとは干渉しません。

Windows

ThunderbirdSetup-0.6.exe インストーラをダブルクリックインストーラを始めます。

MacOS X

Thunderbird の圧縮ディスクイメージをダブルクリックしてマウントします。すでにブラウザーがこのディスクイメージを展開、マウントしているかもしれません。 Thunderbird 0.6 ディスクイメージをダブルクリックして Thunderbird アプリケーションをハードディスクにドラッグします。Dock に表示させたい場合は Thunderbird のアイコンを Dock にドラッグします。

その他のプラットフォーム

圧縮されたアーカイブを展開し thunderbird を実行します。

Thunderbird のアンインストール

Thunderbird をアンインストールするには次の手順に従ってください：

Windows

スタート メニューから コントロールパネル を選択します。 コントロールパネルが表示されたら プログラムの追加と削除 をダブルクリックします。 リストの中の "Mozilla Thunderbird (0.6.)" を見つけて 削除 をクリックしてアンインストールします。

MacOS X

Thunderbird アプリケーションをゴミ箱にドラッグします。

その他のプラットフォーム

thunderbird ファルダを削除します。

既知の問題

以下は Mozilla Thunderbird 0.6 で判明している主な問題のリストです。これらの問題に関しては Bugzilla にバグとして報告しないでください。また、新たな問題に関してバグを報告する前に、このリストで照会してください (このリストは更新されます) 。

  • 重要 Thunderbird の前のバージョンを使用していて、テーマもしくは機能拡張をインストールしている場合は，以下の手順に従ってください。さもないと Thunderbird は正しく動作しないかもしれません。まず プロファイル の場所 (和訳) を 見つけてください。"chrome" と名付けられたサブディレクトリーがあるはずです。そして "chrome" 内のすべてを削除します。この操作がメールのデータや設定に影響を与えることはありません。その後に 0.6 と互換性のあるバージョンのテーマや機能拡張を再度インストールする必要があります。
  • 新しいジャンクメールコントロールの性能を最大限引き出すには、このフィルターをまったくの最初から鍛え直すことをお勧めします。 Tools > Junk Mail Controls > Adaptive Filters > Reset Training Data。同じ数の良いメッセージとジャンクメッセージを教え込む必要がありま す。それぞれ数百のメッセージを教え込むことを勧めます。
  • Linux: Thunderbird はブラウザー用の URL をデフォルトのブラウザーに正しく渡せるようになりました。この仕組みに関し てはこちらを御覧ください。
  • HTTP プロキシ認証と Thunderbird。ネットワークが HTTP にプロキシ認証を使用している場合、Thunderbird はプロキシを通して http パーツを持ってこないようにする認証ダイアログの表示に失敗 します。
  • リリース 0.6 では初期設定として LDAP プロトコル・バージョン３を使用しています。LDAP v2 を使用している古い LDAP サーバーにアクセスすると次のエラーが出るでしょう： "initialization failed, error code 2: protocol error"。強制的にバージョン２をサポートさせるには prefs.js にあなたの LDAP ディレクトリー用の次の設定を手動 で書き込んでください：user_pref("ldap_2.servers.**サーバー名**.protocolVersion", "2");
  • デフォルトの辞書は英語のみです。英語以外の辞書を追加するにはイン ストールの説明を参照してください。
  • 学習機能つきジャンクメールフィルターを有効/無効にするオプションがすべてのアカウントに適用できるように見えます (Tools / Junk Mail Controls / Adaptive Filters)。しかしこれはアカウントごとのオプションです。特 定のアカウントにこのオプションを設定するには、'Settings' パネルの 'Account:' ドロップダウンからそのアカウントを選択してか ら 'Adaptive Filters' パネルに移り、このオプションを設定します。そして必要なだけ各アカウントに対してこの作業を繰り返します。
  • 現時点では Mozilla App-Suite のプロファイルを Thunderbird に移行するようにはなっていません。Mozilla プロファイルを手動で (日本語版) 移 行することができます。
  • デフォルトのテーマほぼ完成しつつありますが、まだいくつかのクラシックテーマが残っています。新しいテーマへの移行が済んでいないことに関 するバグは報告しないでください。
  • mailnews.reply_header_authorwrote preference 設定は HTML を受け付けなくなりました。これはユニコードの設定ですので、各国の文字を入力するには UTF-8 エディターを使用することができます。
