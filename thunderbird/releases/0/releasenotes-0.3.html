Thunderbird リリースノート

バージョン 0.3

Mozilla Thunderbird メールクライアントとは、統合された Mozilla アプリ - スイートのメールコンポーネントを 再デザインしたものです。私たちの目的は、Mozilla メールコンポーネントに今ある機能の多くを活用し、シンプルで拡張性のある単体のメールアプリケーションを作成することです。Mozilla Thunderbird プロジェクトのウェブサイトにはさらに多くの情報があります。

Mozilla アプリ - スイートの基礎的なコードが成熟しているおかげで Mozilla Thunderbird はかなり使いものになります；しかしながら、状況的にはこのリリースはまだアルファ版の前段階であり、従って欠陥があることも十分考えられます。こうした 欠陥を見つけるのに役立てるため、開発者たちは喜んでみなさんのフィードバックを受け付けています ( MozillaZine が親切にもホストしてくれている Mozilla Thunderbird forums へのコメントか Bugzilla にバグを報告してください) 。どんなバグでも Bugzilla にバグを報告する前にはこのリリースノートを十分に読んでください。

この三番目のマイルストーンリリースでは、バグの修正とパフォーマンスに焦点が当てられています。私たちはリリース 0.2 で多くの新しい機能を導入しました。そしてこのリリースでは、機能の追加は少なくして安定性により注力したものを望みました。このマイルストーンリリース は最近リリースされた Mozilla 1.5 アプリケーションスイートを基礎にしています。さらにまだ多くの作 業が必要ではありますが、開発者たちはこの最近の進展にわくわくし、コミュニティのみなさんとこの最新の作業成果を分かち合いたいと切望しています。では 楽しんで！

新機能

以下のリストは主に進展した箇所の要約です。詳細は変更履歴を参照してください。
  • 最近修正されたバグ

    添付書類を開く時のダイアログにいくつもの改良が加えられました。この改良には 0.2 でよく見られた、添付書類を開こうとする時に "Open With" オプションが使用できない問題の修正も含まれています。

    返信のテキストがメッセージの一番上になるように設定している場合、署名を返信するメッセージの一番上に置くことができます。

    POP アカウントの設定で、設定した日にち以降に既読のメッセージをサーバーから取り除くことができるようになりました。

    背景画像を持つ新しい文章を読む場合に CPU を１００％占有しないようになりました。

    Windows と Linux のバージョンで Binhex でエンコードされた添付書類を正しく扱えるようになりました。

  • パフォーマンスの改善

    IMAP のパフォーマンスがかなり改善され、0.2 に比べて何倍も速くなりました。

  • 改善された安定性と外観

    このリリースは Mozilla アプリ - スイート 1.5 の最終ブランチを基礎にしていて、Mozilla 1.5 に加えられた安定化作業のすべてを含んでいます。

ダウンロード、インストールとアンインストール

初 めてインストールする場合：このリリースには正式なインストーラはありません。下記のインストールの説明は、単一の 圧縮されたプログラムファイルをあなたがダウンロードすることを想定して書かれています。このプログラムファイルはハードディスクに手動で展開しなくては なりません。このようなインストール方法は、難しくはありませんが展 開するためのユーティリティを必要とします。 (詳しい使い方はそのユーティリティのヘ ルプを参照してください)

下記にリストされている提供されたビルドの中には、インストールのプロセスを簡単にするかもしれない自己解凍型やインストーラとしてパッケージされたもの があるかもしれません。もう一つの方法としては、mozdev.org にある Thunderbird Installer を試してみてもよいでしょう。

アップグレードする場合： 古 いバージョンからのプログラムファイルを含むディレクトリーに Mozilla Thunderbird をインストールしないでください。古いバージョンに上書きすると問題を引き起こします。以前にインストールしたディレクトリーを再利用するには、 そのディレクトリーを削除して再度作り直すか、空にするか、移動するか、あるいは違う名前を付けます。この手順を無視した場合には絶対に Bugzilla にバグを登録しないでください。

このプログラム自体のディレクトリーにはプロファイル情報は含まれていません。従って新たにインストールしても、今あるアカウント、アカウントの設定、オプ ション、メール、そしてニュースメッセージが損なわれることはありません。このリリースではプロファイルを変更しなくても正しく作動します。好奇心の強い 人／微調整を望む人／FUD に捕われている人、これらの人たちは Mozilla Thunderbird Help (日本語版) を参考にプロファイル の場所 (和訳) を 見つけてください。

インストール：
注意：Mozilla Thunderbird は専用のユーザープロファイル情報を作成します。もし無くても Mozilla アプリ - スイートのプロファイルとは干渉しません。

ステップ１：下記にあるリンクから自分のオペレーティングシステムと環境にあったものをクリックし、バイナリをダウンロードします。
ステップ２a：アップグレードの場合はインストールするディレクトリーを準備するか、新しいディレクトリーを使用するようにしてください (上記の 黄色いアップグ レードの部分を 参照してください) 。
ステップ２b：展 開するためのユーティリティを使用してハードディスクの空のディレクトリーにファイルを展開します。
ステップ３：展開したディレクトリーに移動し、 thunderbird.exe を実行します。
ステップ４：必要であれば、素早くアクセスするためにショートカットやシンボリックリンクを作成します。

アンインストール：
注意：ステップ２を実行しなければプロファイルを残しておくことができます。これ は将来再び Mozilla Thunderbird をインストールして使用する際に利用ですることができます。

ステップ１：Mozilla Thunderbird のファイルを展開したプログラムディレクトリーを削除します。
ステップ２：Mozilla Thunderbird のプロファイルディレクトリーを削除します。このステップを実行すると、プロファイルにローカルで保存されているすべてのメールが削除されます。
ステップ３：作成したショートカットやシンボリックリンクを削除します。

既知の問題

以下は Mozilla Thunderbird 0.3 で判明している問題の顕著なもののリストです。これらの問題に関しては Bugzilla にバグとして報告しないでください。また、新たな問題に関してバグを報告する前に、このリストで照会してください (このリストは更新されます) 。

  • デフォルトのテーマがまだ完全ではありません。未だに Mozilla 用のクラッシックテーマのアイコンを使用しています。 Thunderbird のテーマはこれからも改良され続けて、1.0 リリースまでには完全なものになるでしょう。
  • デフォルトでは POP アカウントはサーバーにメッセージを残さないようになりました。これは Tools / Account Settings / Server Settings で変更できます。
  • デフォルトの辞書は英語のみです。英語以外の辞書を追加するにはイン ストールの説明を参照してください。
  • 学習機能つきジャンクメールフィルターを有効/無効にするオプションがすべてのアカウントに適用できるように見えます (Tools / Junk Mail Controls / Adaptive Filters)。しかしこれはアカウントごとのオプションです。特 定のアカウントにこのオプションを設定するには、'Settings' パネルの 'Account:' ドロップダウンからそのアカウントを選択してか ら 'Adaptive Filters' パネルに移り、このオプションを設定します。そして必要なだけ各アカウントに対してこの作業を繰り返します。
  • 'Download Rest of Message' リンクは機能しません。これは、該当する POP メールのアカウント設定で '大きな' メッセージをサーバーに残すよう設定している場合に、大きすぎて一部分だけしかダウンロードされなかったメッセージに対してのみ機能するものです。解決法としてはより大きなメッセージもダウンロードできるように POP アカウントを設定することです。
  • 現時点では Mozilla プロファイルを Thunderbird に移行するようにはなっていません。Mozilla プロファイルを手動で (日本語版) 移 行することもできますが、私たちとしてはおすすめしません。たとえ Thunderbird をプレビューとして試用してみたい場合でも新しいプロファイ ルで使用することを強くすすめます (そのプロファイルは後のリリースをプレビューする時に再び使用することができます) 。
  • メールの３ペイン内でのプログレスバーは表示されません。
  • Windows：Windows のインターネットオプションを使用して Mozilla Thunderbird をデフォルトのメールクライアントに設定する方法はあてになりません。Thunderbird の Tools / Options General から設定することをおすすめします。
  • Linux：オペレーティングシステムとの統合がまだ済んでいません。メールメッセージ内の HTTP URL をクリックしても Linux のデフォルトブラウザーを立ち上げることができません。この問題は次のリリースで解決したいと考えています。この問題が Linux ユーザーにとってどれだけ重要な問題かは理解していますので、あらかじめお詫びしておきます
  • Linux：Mark All Read のキーボードショートカットが動きません。MacOSX：Thunderbird の開いているウインドウのすべてを閉じようとすると、部分的に使用できない変なメニューが現れます。これは Mozilla アプリ - スイートのインターフェース要素が表出してしまうためで、大抵はただうっとうしいだけのものです。この '隠れたウインドウ' をどうにかする必要があります。できれば四番目のマイルス トーンビルドまでには何とかするつもりです
  • MacOSX：開いているアプリケーションウインドウを切り替えることのできる 'ウインドウ' メニュー項目がメニューバーにまだありません。これも近い将 来に修正したいと考えています。
