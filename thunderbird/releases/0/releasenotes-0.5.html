Thunderbird リリースノート

バージョン 0.5

Mozilla Thunderbird は、総合的な Mozilla アプリ - スイートのメールコンポーネントを 再デザインしたものです。その目的は Mozilla メールコンポーネントに今ある機能の多くを活用し、シンプルで拡張性のある単体のメールアプリケーションを作成することです。 Thunderbird メール プロダクトのページにはさらに多くの情 報があります。

Mozilla アプリ - スイートの基礎的なコードが成熟しているおかげで Mozilla Thunderbird はかなり使いものになります；しかしながら、このリリースはまだプレビューリリースであり、従って欠陥があることも十分考えられます。こうした 欠陥を見つけるのに役立てるため、開発者たちは喜んでみなさんのフィードバックを受け付けています ( MozillaZine が親切にもホストしてくれている Mozilla Thunderbird forums へのコメントか Bugzilla にバグを報告してください) 。どんなバグでも Bugzilla にバグを報告する前にはこのリリースノートを十分に読んでください。

この五番目のマイルストーンでは安定性とバグのフィックスに焦点が当てられていて、最近リリースされた Mozilla 1.6 アプリケーションスイートをベースにしています。さらにまだ多くの作 業を必要としていますが、開発者たちはこの最近の進展にわくわくし、コミュニティのみなさんとこの最新の作業成果を分かち合いたいと切望しています。では 楽しんでください！

最新情報

このリリースのハイライトは以下のとおりです：
  • 新機能

    各メールアカウントに複数の ID が持てる機能をサポートしました。これによって同じアカウントで利用する複数のメールアドレスを持ちやすくなります。使い方についてはこちらを御覧ください。

    Thunderbird 0.5 では POP3 と SMTP 用に、新しいクロスプラットフォームな NTLM 認証を使用したセキュア・パスワード認証を利用できます。

    フォルダー コラムを Mozilla 1.x のスタイルにするオプションがつきました (Tools / Options / Advanced / General Settings)。

     0.5 用の新しい Palm Sync Address book conduit が利用可能になりました。hot sync コンジット設定を変更することによって PC -> Palm もしくは Palm -> PC の一方向同期をすることができる ようになりました。Palm カテゴリーと同期をとる際、Thunderbird のアドレスブックに重複したカードができないようにしました。また、初期の同期に関して多くの改良がなされました

    0.5 用の新しく改良されたバージョンの offline extension (オフライン機能拡張) が利用可能になりました。古い機能拡張のアンインストールの方法についてこのページのインストールをよく読んでください。

    スペルチェッカーが改良されました。新しい US 辞書を含んでいます。

    新規メール作成ウインドウのアドレス部分に、表計算ソフトから名前やアドレスをペーストできるようになりました。

    Netscape 4.x からのプロフィールの移動が改良されました

  • 修正されたバグ

    メールを読んでいる時に新しいメールを受信しても、読んでいるメールの一番上に勝手にスクロールしないようになりました。

    IMAP の添付書類を保存する時に、サーバーからその添付書類を再ダウンロードしないようにしました。

    添付書類を保存する際、独立した進行状況ウインドウが表示されます。

    Sent Folder (送信済みフォルダー) にコピーする時、進行状況ウインドウに進行状況が表示されます。

    添付書類を保存 / 開く時、進行状況ダイアログが詳細な進行状況情報を表示します。

    削除されたかほかのフォルダーに移された IMAP のメッセージを「既読」として扱うことにしました。これによって、Inbox 以外のフォルダーで新しいメッセージをチェックする際に未読のメッセージ数が間違ってカウントされないようなります。

    LDAP 検索がディレクトリー検索フィルタープロパティを参照するようになりました。

    メール作成ウインドウに Microsoft Excel からデータの単一セルをペーストできるようになりました。

    IMAP サブスクリプション・モデルを使用時に三階層目の IMAP フォルダーが表示されない問題が解決されました。

    Tools (ツール) / Options (オプション) / Attachments (添付書類) / の添付書類フォルダーの設定を記憶するようになりました。

    Linux のビルドで JAVA を必要とする HTML メッセージを表示させてもクラッシュしないようにしました。

    OSX のクライアントからの apple/double でエンコードされた添付書類の取り扱いが改善されました。

    受信したメールが "ジャンク" だった場合、新規メール受信のお知らせは表示されません。

    "Remove from server after x days (X 日後にサーバーから削除) " オプションが POP3 メールアカウント用に実装されました。

  • パフォーマンスの改善

    フォルダーの読み込みが早くなりました。

  • 安定性の改善

    このリリースは Mozilla アプリ-スイート 1.6 の最終ブランチをベースにしていて、Mozilla 1.6 に盛り込まれた安定化に関するすべての作業成果を含んでいます。

ダウンロード、インストールとアンインストール

初 めてインストールする場合：このリリースには正式なインストーラはありません。下記のインストールの説明は、単一の 圧縮されたプログラムファイルをあなたがダウンロードすることを想定して書かれています。このプログラムファイルはハードディスクに手動で展開しなくては なりません。このようなインストール方法は、難しくはありませんが展 開するためのユーティリティを必要とします。 (詳しい使い方はそのユーティリティのヘ ルプを参照してください)

下記にリストされている提供されたビルドの中には、インストールのプロセスを簡単にするかもしれない自己解凍型やインストーラとしてパッケージされたもの があるかもしれません。もう一つの方法としては、mozdev.org にある Thunderbird Installer を試してみてもよいでしょう。

アッ プグレードする場合： 前のバージョンのプ ログラムファイルを含むディレクトリーに Mozilla Thunderbird をインストールしないでください。前のバージョンに上書きすると問題を引き起こす可能性があります。以前にインストールしたディレクトリーを再利 用するには、 そのディレクトリーを削除して再度作り直すか、空にするか、移動するか、あるいは違う名前を付けます。この手順を無視した場合には絶対に Bugzilla にバグを登録しないでください。

このプログラム自体のディレクトリーにはプロファイル情報は含まれていません。従って新たにインストールしても、今あるアカウント、アカウントの設定、オプ ション、メール、そしてニュースメッセージが損なわれることはありません。このリリースではプロファイルを変更しなくても正しく作動します。

重要： Thunderbird の前のバージョンを使用していて、テーマもしくは機能拡張をインストールしている場合は，以下の手順に従ってください。さもないと Thunderbird は正しく動作しないかもしれません。まず プロファイル の場所 (和訳) を 見つけてください。"chrome" と名付けられたサブディレクトリーがあるはずです。そして "chrome" 内のすべてを削除します。この操作がメールのデータや設定に影響を与えることはありません。

インストール：
注意：Mozilla Thunderbird は専用のユーザープロファイル情報を作成します。もし無くても Mozilla アプリ - スイートのプロファイルとは干渉しません。

ステップ１：下記にあるリンクから自分のオペレーティングシステムと環境にあったものをクリックし、バイナリをダウンロードします。
ステップ２a：アップグレードの場合はインストールするディレクトリーを準備するか、新しいディレクトリーを使用するようにしてください (上記の 黄色いアップグ レードの部分を 参照してください) 。
ステップ２b：展 開するためのユーティリティを使用してハードディスクの空のディレクトリーにファイルを展開します。
ステップ３：展開したディレクトリーに移動し、 thunderbird.exe を実行します。
ステップ４：必要であれば、素早くアクセスするためにショートカットやシンボリックリンクを作成します。

アンインストール：
注意：ステップ２を実行しなければプロファイルを残しておくことができます。これ は将来再び Mozilla Thunderbird をインストールして使用する際に利用ですることができます。

ステップ１：Mozilla Thunderbird のファイルを展開したプログラムディレクトリーを削除します。
ステップ２：Mozilla Thunderbird のプロファイルディレクトリーを削除します。このステップを実行すると、プロファイルにローカルで保存されているすべてのメールが削除されます。
ステップ３：作成したショートカットやシンボリックリンクを削除します。

既知の問題

以下は Mozilla Thunderbird 0.5 で判明している主な問題のリストです。これらの問題に関しては Bugzilla にバグとして報告しないでください。また、新たな問題に関してバグを報告する前に、このリストで照会してください (このリストは更新されます) 。

  • *重要*： Thunderbird の前のバージョンを使用していて、テーマもしくは機能拡張をインストールしている場合は，以下の手順に従ってください。さもないと Thunderbird は正しく動作しません。まず プロファイル の場所 (和訳) を 見つけてください。"chrome" と名付けられたサブディレクトリーがあるはずです。そして "chrome" 内のすべてを削除します。この操作がメールのデータや設定に影響を与えることはありません。その後，0.5 と互換性のある使用したいテーマ/機能拡張の バージョンを再びインストールします。
  • Linux：Thunderbird はブラウザーの URL を正しくデフォルトのブラウザーに渡します。この設定方法についてはこちらを参照してください。
  • デフォルトの辞書は英語のみです。英語以外の辞書を追加するにはイン ストールの説明を参照してください。
  • 学習機能つきジャンクメールフィルターを有効/無効にするオプションがすべてのアカウントに適用できるように見えます (Tools / Junk Mail Controls / Adaptive Filters)。しかしこれはアカウントごとのオプションです。特 定のアカウントにこのオプションを設定するには、'Settings' パネルの 'Account:' ドロップダウンからそのアカウントを選択してか ら 'Adaptive Filters' パネルに移り、このオプションを設定します。そして必要なだけ各アカウントに対してこの作業を繰り返します。
  • 何人かのユーザーからジャンクメールコントロールが動作しないとの報告を受けました。まず最初に、問題になっているアカウントで学習機能つき ジャンクメールフィルターが有効になっているか確認してください。有効になっていてもこの機能が動作しない場合は、プロファイルから training.dat を取り除き、再度このフィルターに学習させてください。
  • 現時点では Mozilla プロファイルを Thunderbird に移行するようにはなっていません。Mozilla プロファイルを手動で (日本語版) 移 行することができます。
  • デフォルトのテーマほぼ完成しつつありますが、まだいくつかのクラシックテーマが残っています。新しいテーマへの移行が済んでいないことに関 するバグは報告しないでください。
  • Linux：Mark All Read のキーボードショートカットが動きません。
  • Mac OS X: Thunderbird は他のアプリケーションから mailto/news URL を開くことができません。
  • Mac OS X: 特定の文字コード (韓国語等) での添付書類名が正しくレンダリングされないことがあります。
  • mailnews.reply_header_authorwrote 設定は HTML を受け付けなくなりました。これはユニコードの設定ですので、各国の文字を入力するには UTF-8 エディターを使用することができます。
  • インラインイメージ入りのメッセージを閲覧する場合に，コンテキストメニュー項目の "View Image" を選択すると小さな空のウインドウが表示されてしまいます。
