Thunderbird リリースノート

バージョン 0.7

Thunderbird はフル装備の E-メール / ニュースグループクライアントで、かつてないほど安全で素早く、そして簡単にメールを扱うことができます。Thunderbird についてのより詳しい情報はこちらを参照してください。

Mozilla の基礎的なコードが成熟しているおかげで Thunderbird は十分使用できる状態にあります。しかしながら、このリリースはまだテクノロジー・プレビューであり、従って欠陥があることも十分考えられ ます。詳しい情報は License Agreement を参照してください。

最新情報

このリリースのハイライトは以下のとおりです：
  • より小さくより速く

    Windows 用インストーラのダウンロードサイズが 5.9MB になりました。Windows、Linux そして OS X 上での性能が大幅に向上しました！

  • 新しいテーマ/機能拡張マネージャー

    Thunderbird を他のメールクライアントから引き離すたくさんのアドオンを管理/アップデートするための効率的で安全な手段を提供します。

  • クラッシュ分析ツール (Talkback)

    みなさんを助けるために私たちを助けてください！ユーザーは Mozilla Talkback でクラッシュの報告を送信することができます。これによって私たちは Thunderbird でどんなクラッシュが最も頻繁に起こっているのか容易に把握することができます。

  • その他の新機能...

    vCards 閲覧のための新しいユーザーインターフェース

    USB 機器からの作動のサポートもする新しいプロファイルマネージャー

    メッセージ引用部分の色づけ

    アカウント毎の複数 ID 作成用のユーザーインターフェースをサポートするようになりました。これにより、最終的に同じアカウントに入る複数のメールアドレスを簡単に持つことができ ます。

  • 最近フィックスされたバグ

    Mac OS X：スリープモードから復帰する際に Thunderbird がハングするバグを解消。

    Mac OS X：Thunderbird のウインドウが何も開いていない時に Dock 上の Thunderbird のアイコンをクリックするとメール主ウインドウが開くようになりました。

    SSL を通して IMAP メールを読んでいる時にときどきハングするバグを解消。

    バージョン３に加え、以前の LDAP バージョン 2 サーバーに対する LDAP サポートが再び動作するようになりました

    Thunderbird は未加工のスペースを件名 (Subject) に含む mailto URL を扱えるようになりました

  • 0.7.1 でさらに改善された箇所

    プロファイルのディレクトリーパスに非アスキー文字が含まれている場合に、そのプロファイルを認識できない問題を解決しました

    Mac OS X：Thunderbird 0.7 は 0.6 のプロファイルを認識していませんでした。

    Windows のシステムトレイに新規メールお知らせアイコンが表示されないことのある問題を解決しました

    機能拡張システムを改良しました

ダウンロード、インストールとアンインストール

アップグレードする場合： 前 のバージョンのプ ログラムファイルを含むディレクトリーに Mozilla Thunderbird をインストールしないでください。前のバージョンに上書きすると問題を引き起こす可能性があります。以前にインストールしたディレクトリーを再利 用するには、 そのディレクトリーを削除して再度作り直すか、空にするか、移動するか、あるいは違う名前を付けます。この手順を無視した場合には絶対に Bugzilla にバグを登録しないでください。

このプログラム自体のディレクトリーにはプロファイル情報は含まれていません。従って新たにインストールしても、今あるアカウント、アカウントの設定、オプ ション、メール、そしてニュースメッセージが損なわれることはありません。このリリースではプロファイルを変更しなくても正しく作動します。

Thunderbird のインストール

Thunderbird のインストーラあるいは圧縮アーカイブをダウンロードしたら次の説明に従ってインストールしてください：

注意： Mozilla Thunderbird は専用のユーザープロファイル情報を作成します。もし無くても Mozilla アプリ - スイートのプロファイルとは干渉しません。

Windows

ThunderbirdSetup-0.7.exe インストーラをダブルクリックインストーラを始めます。

MacOS X

Thunderbird の圧縮ディスクイメージをダブルクリックしてマウントします。すでにブラウザーがこのディスクイメージを展開、マウントしているかもしれません。 Thunderbird 0.7 ディスクイメージをダブルクリックして Thunderbird アプリケーションをハードディスクにドラッグします。Dock に表示させたい場合は Thunderbird のアイコンを Dock にドラッグします。

その他のプラットフォーム

圧縮されたアーカイブを展開し thunderbird を実行します。

Thunderbird のアンインストール

Thunderbird をアンインストールするには次の手順に従ってください：

Windows

スタート メニューから コントロールパネル を選択します。 コントロールパネルが表示されたら プログラムの追加と削除 をダブルクリックします。 リストの中の "Mozilla Thunderbird (0.7.)" を見つけて 削除 をクリックしてアンインストールします。

MacOS X

Thunderbird アプリケーションをゴミ箱にドラッグします。

その他のプラットフォーム

thunderbird ファルダを削除します。

注意すべき変更点

テーマと機能拡張

0.6 リリースで起こった種類の問題 (アプリケーションが起動しない、あるいはテーマが壊れた) を避けるために、0.7 を初めて起動すると、以前にインストールしたすべての機能拡張が自動的に停止され、テーマはデフォルトに戻されます。そしてそのあとに、 Thunderbird 0.7 と互換性のある機能拡張の新しいバージョンを探します。Thunderbird 0.7 以降はこうしたアップデートの作業はよりシームレスになるはずです。

既知の問題

以下は Mozilla Thunderbird 0.7 で判明している主な問題のリストです。これらの問題に関しては Bugzilla にバグとして報告しないでください。また、新たな問題に関してバグを報告する前に、このリストで照会してください (このリストは更新されます) 。

  • 0.7 へのアップグレードのさい、'offline mode (オフライン・モード) ' から変更できなくなることがあります。こうなると送信メッセージが未送信フォルダーに追加され、新しいメールのダウンロードに失敗してしまいます。オフライ ン機能拡張を一緒にインストールしているのなら、アプリケーションのステータスバーにあるオフライン・インジケータを使用してオンライン・モードに戻しま す。もしくはプロファ イルの prefs.js ファイルから network.online 設定を削除します。
  • Thunderbird 0.7 がアクセス権の制限されている場所におかれている場合、その場所に対するアクセス権を持ったユーザーが最初に起動してください。そうすることですべての初期 起動ファイルが生成されます。アクセス権の制限されたユーザーが最初に起動すると、これらのファイルが正しく生成されない恐れがあり、 Thunderbird が再起動の無限ループに入ってしまうかもしれません。この問題は将来のリリースでフィックスされます。
  • Linux: Thunderbird はブラウザー用の URL をデフォルトのブラウザーに正しく渡せるようになりました。その方法に関し てはこちらを御覧ください。
  • HTTP プロキシ認証と Thunderbird。ネットワークが HTTP にプロキシ認証を使用している場合、Thunderbird はプロキシを通して http パーツを持ってこないようにする認証ダイアログの表示に失敗 します
  • デフォルトの辞書は英語のみです。英語以外の辞書を追加するにはイン ストールの説明を参照してください。
  • 学習機能つきジャンクメールフィルターを有効/無効にするオプションがすべてのアカウントに適用できるように見えます(Tools / Junk Mail Controls / Adaptive Filters)。しかしこれはアカウントごとのオプ ションです。特 定のアカウントにこのオプションを設定するには、'Settings' パネルの 'Account:' ドロップダウンからそのアカウントを選択してか ら 'Adaptive Filters' パネルに移り、このオプションを設定します。そして必要なだけ各アカウントに対してこの作業を繰り返します。
  • 現時点では Mozilla App-Suite のプロファイルは Thunderbird に移行されません。手動でならMozilla プロファイルを (日本語版) 移 行することができます。
  • デフォルトのテーマほぼ完成しつつありますが、まだいくつかのクラシックテーマが残っています。新しいテーマへの移行が済んでいないことに関 するバグは報告しないでください。
  • 送信メールのデフォルト文字コード設定に問題が生じることがあります。
