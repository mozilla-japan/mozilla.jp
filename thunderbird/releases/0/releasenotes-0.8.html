Thunderbird リリースノート

バージョン 0.8

Thunderbird はフル装備の E-メール / ニュースグループクライアントで、かつてないほど安全で素早く、そして簡単にメールを扱うことができます。Thunderbird についての より詳しい情報はこちら を参照してください。

Mozilla の基礎的なコードが成熟しているおかげで Thunderbird は十分使用できる状態にあります。しかしながら、このリリースはまだ テクノロジープレビュー であり、従って欠陥があることも十分考えられます。詳しい情報は ライセンス規約 を参照してください。

このリリースノートでは以下のトピックを扱っています：

  • 最新情報
  • システム動作環境
  • ダウンロード、インストールとアンインストール

  • ユーザーサポートとフィードバック

どんなバグでも Bugzilla にバグを報告する前にこのリリースノートと バグの登録方法を読んでください。Thunderbird に関するフィードバックを歓迎します。MozillaZine が快くホストしてくれている Mozilla Thunderbird サポートフォーラム に参加してみてください。

最新情報

このリリースのハイライトは以下のとおりです：

  • 共通受信トレイ

    POP3 ユーザーは、すべての POP3 アカウントをローカルフォルダー内の一つの 共通受信トレイ にまとめることができるようになりました。

  • 総合的なデータ移行

    Mozilla 1.x Suite、Outlook Express、Outlook や Eudora など、一般的なメールアプリケーションの各種設定、メールフォルダー、アドレスブックなどの E-メールデータを移行させることができるようになり、Thunderbird への乗り換えがかつてないほど簡単になりました。

  • RSS の統合

    Thunderbird に RSS リーダー機能が組み込まれ、RSS をサポートするニュースやウェブログを簡単に購読できるようになりました。

  • 改善されたプライバシーコントロール

    あなたのプライバシーを守るために、Thunderbird は未知の送信者からのメールに含まれるリモートイメージリクエストを自動的にブロックするようになりました。

  • 改善されたクイック検索

    メールの管理をさらに簡単にします。クイック検索は、メッセージ本体の検索機能を含むたくさんの種類の検索条件をサポートするようになりました。また、クイック検索した語句をメッセージ本体内で強調表示することもできます。

  • その他の新機能

    保存されたメールアカウントパスワードを暗号化するマスターパスワードの使用をサポート。

    Linux GNOME のユーザーは Thunderbird をデフォルトのメールクライアントに設定できるようになりました (ツール > オプション > 一般設定 (Tools > Options > General) )

    ネットワークが HTTP にプロキシ認証を使用している場合、Thunderbird はプロキシ認証を正しく指示するようになりました。

    ここに書ききれないほどたくさんのバグフィックス！

ダウンロード、インストールとアンインストール

アップグレードする場合： 前のバージョンのプログラムファイルを含むディレクトリーに Mozilla Thunderbird をインストールしないでください。前のバージョンに上書きすると問題を引き起こす可能性があります。以前にインストールしたディレクトリーを再利用するには、そのディレクトリーを削除して再度作り直すか、空にするか、移動するか、あるいは違う名前を付けます。この手順を無視した場合には絶対に Bugzilla にバグを登録しないでください。

このプログラム自体のディレクトリーにはプロファイル情報は含まれていません。従って新たにインストールしても、今あるアカウント、アカウントの設定、オプション、メール、そしてニュースメッセージが損なわれることはありません。このリリースではプロファイルを変更しなくても正しく作動します。

Thunderbird のインストール

Thunderbird のインストーラあるいは圧縮アーカイブをダウンロードしたら次の説明に従ってインストールしてください：

注意：Mozilla Thunderbird は、プロファイル情報がなければ Thunderbird 専用のユーザープロファイル情報を作成します。Mozilla アプリ - スイートのプロファイルとは干渉しません。

Windows

ThunderbirdSetup-0.8.exe インストーラをダブルクリックインストーラを始めます。

MacOS X

Thunderbird の圧縮ディスクイメージをダブルクリックしてマウントします。すでにブラウザーがこのディスクイメージを展開、マウントしているかもしれません。Thunderbird 0.8 ディスクイメージをダブルクリックして Thunderbird アプリケーションをハードディスクにドラッグします。Dock に表示させたい場合は Thunderbird のアイコンを Dock にドラッグします。

その他のプラットフォーム

圧縮されたアーカイブを展開し thunderbird を実行します。

Thunderbird のアンインストール

Thunderbird をアンインストールするには次の手順に従ってください：

Windows

スタート メニューから コントロールパネル を選択します。コントロールパネルが表示されたら プログラムの追加と削除 をダブルクリックします。 リストの中の "Mozilla Thunderbird (0.8.)" を見つけて 削除 をクリックしてアンインストールします。

MacOS X

Thunderbird アプリケーションをゴミ箱にドラッグします。

その他のプラットフォーム

thunderbird ファルダを削除します。

既知の問題

以下は Mozilla Thunderbird 0.8 で判明している主な問題のリストです。これらの問題に関しては Bugzilla にバグとして報告しないでください。また、新たな問題に関してバグを報告する前に、このリストで照会してください (このリストは更新されます)。

  • Thunderbird 0.8 がアクセス権の制限されている場所におかれている場合、その場所に対するアクセス権を持ったユーザーが最初に起動してください。そうすることですべての初期起動ファイルが生成されます。アクセス権の制限されたユーザーが最初に起動すると、これらのファイルが正しく生成されない恐れがあり、Thunderbird が再起動の無限ループに入ってしまうかもしれません。この問題は将来のリリースでフィックスされます。
  • RSSフィードがユーザー名とパスワードを要求する場合、RSS リーダー機能は使用できません。
  • アカウントマネージャーでは複数の　RSS アカウントを作成できません。
  • Linux と OS X で RSS フィードを含むフォルダーを削除するには RSS 購読ダイアログ (subscription dialog) を使用します。Windows ユーザーはフォルダーペインから削除や名称変更を直接行うことができます。
  • Linux: Thunderbird は初期設定のままではブラウザー用の URL をデフォルトのブラウザーに正しく渡せません。これを行うには こちらを参照 してください。
  • デフォルトの辞書は英語のみです。英語以外の辞書を追加するにはインストールの説明を参照してください。
  • Windows のユーザーは Thunderbird 用の Palm Sync extension をインストールすることによって Palm 製品とアドレス帳を同期させることができます。まずこの拡張機能のファイルをローカルに保存し、それから Thunderbird の拡張機能ダイアログを使用してインストールします。この拡張機能は拡張機能の一覧には表示されません。
  • 学習機能つき迷惑メールフィルターを有効/無効にするオプションがすべてのアカウントに適用できるように見えます(Tools / Junk Mail Controls / Adaptive Filters)。しかしこれは アカウントごとの オプションです。特定のアカウントにこのオプションを設定するには、'Settings' パネルの 'Account:' ドロップダウンからそのアカウントを選択してから 'Adaptive Filters' パネルに移り、このオプションを設定します。そして必要なだけ各アカウントに対してこの作業を繰り返します。
  • デフォルトのテーマほぼ完成しつつありますが、まだいくつかのクラシックテーマが残っています。新しいテーマへの移行が済んでいないことに関するバグは報告しないでください。
