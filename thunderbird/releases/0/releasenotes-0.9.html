Thunderbird リリースノート

バージョン 0.9

Thunderbird はフル装備の E-メール / RSS / ニュースグループクライアントで、かつてないほど安全で素早く、そして簡単にメールを扱うことができます。Thunderbird についての より詳しい情報はこちら を参照してください。

Mozilla の基礎的なコードが成熟しているおかげで Thunderbird はデフォルトメールクライアントとして十分使用できる状態にあります。しかしながら、このリリースはまだ テクノロジープレビュー であり、従って欠陥があることも十分考えられます。詳しい情報は ライセンス規約 を参照してください。

このリリースノートには、Thunderbird 0.9 に関する 最新情報、ダウンロード と インストール方法、既知の問題、ユーザーサポートとフィードバック などの情報が載っています。

どんなバグでも Bugzilla にバグを報告する前にこのリリースノートと バグの登録方法を読んでください。Thunderbird に関するフィードバックを歓迎します。MozillaZine が快くホストしてくれている Mozilla Thunderbird サポートフォーラム に参加してみてください。

最新情報

このリリースのハイライトは以下のとおりです：

  • 検索フォルダー

    検索フォルダー は、これまでに設定された検索条件に基づいてメッセージを表示します。メッセージにフィルターをかけて新しいフォルダーに移動する代わりに、例えば、特定の人から 30 日以内に受け取ったすべてのメッセージをリストアップした検索フォルダーを作成できます。それらのメッセージが別のフォルダーやサブフォルダーに保存されていても、ひとつのフォルダーにまとめて並べることができるのです。

  • メッセージのグループ化

    日付、送信者、優先度、カスタムラベルなどの属性で、フォルダー内の メッセージをグループ化 できるようになりました。例えば、日付でグループ化されたフォルダーでは、メッセージが今日、昨日、先週といった基準で分類され、メッセージリストの中で自動的にグループ化されます ([表示] > [並べ替え順序] > [並べ替えでグループ化])

  • その他の新機能

    メッセージリスト内に表示される添付ファイルマークが、それぞれのメッセージを選択したときではなく、リストの表示と同時に表示されるようになりました。

    POP3 の 共通受信トレイ サポートが改良されました。

    Thunderbird 0.8 で導入されたクイック検索バーで、検索語が入力されたときにクリアボタンが表示されるようになりました。

    アカウントマネージャーでローカルフォルダーのパスを変更できなかった Thunderbird 0.8 のリグレッションを修正しました。

    オフライン関連のよくある問題の修正を含め、オフラインサポートが改良されました。

    アドレス帳に載っていない送信者からのメッセージに含まれるリモートコンテンツをブロックするプライバシー管理が改良されました。

    メッセージペインの中で長い添付ファイル名が短縮されなくなりました。

    その他、ここにはとても載せきれないたくさんのバグの修正

Thunderbird のインストール

Thunderbird のインストーラあるいは圧縮アーカイブをダウンロードしたら次の説明に従ってインストールしてください：

注意：Mozilla Thunderbird は、プロファイル情報がなければ Thunderbird 専用のユーザープロファイル情報を作成します。これまでに Mozilla Suite をインストールしてあった場合でも、Thunderbird は Mozilla Suite のプロファイルとは干渉しません。

Windows

Thunderbird Setup 0.9.exe インストーラをダブルクリックインストーラを始めます。

MacOS X

Thunderbird の圧縮ディスクイメージをダブルクリックしてマウントします。すでにブラウザーがこのディスクイメージを展開、マウントしているかもしれません。Thunderbird 0.9 ディスクイメージをダブルクリックして Thunderbird アプリケーションをハードディスクにドラッグします。Dock に表示させたい場合は Thunderbird のアイコンを Dock にドラッグします。

その他のプラットフォーム

圧縮されたアーカイブを展開し thunderbird を実行します。

Thunderbird のアンインストール

Thunderbird をアンインストールするには次の手順に従ってください：

Windows

スタート メニューから コントロールパネル を選択します。コントロールパネルが表示されたら プログラムの追加と削除 をダブルクリックします。 リストの中の "Mozilla Thunderbird (0.9.)" を見つけて 削除 をクリックしてアンインストールします。

MacOS X

Thunderbird アプリケーションをゴミ箱にドラッグします。

その他のプラットフォーム

thunderbird ファルダを削除します。

既知の問題

以下は Mozilla Thunderbird 0.9 で判明している主な問題のリストです。これらの問題に関しては Bugzilla にバグとして報告しないでください。また、新たな問題に関してバグを報告する前に、このリストで照会してください (このリストは更新されます)。

  • Thunderbird 0.9 がアクセス権の制限されている場所におかれている場合、その場所に対するアクセス権を持ったユーザーが最初に起動してください。そうすることですべての初期起動ファイルが生成されます。アクセス権の制限されたユーザーが最初に起動すると、これらのファイルが正しく生成されない恐れがあり、Thunderbird が再起動の無限ループに入ってしまうかもしれません。この問題は将来のリリースでフィックスされます。
  • RSS フィードがユーザー名とパスワードを要求する場合、RSS リーダー機能は���用できません。
  • アカウントマネージャーでは複数の　RSS アカウントを作成できません。
  • Linux: Thunderbird は初期設定のままではブラウザー用の URL をデフォルトのブラウザーに正しく渡せません。これを行うには こちらを参照 してください。
  • デフォルトの辞書は英語のみです。英語以外の辞書を追加するにはインストールの説明を参照してください。
  • Windows のユーザーは Thunderbird 用の Palm Sync extension をインストールすることによって Palm 製品とアドレス帳を同期させることができます。まずこの拡張機能のファイルをローカルに保存し、それから Thunderbird の拡張機能ダイアログを使用してインストールします。この拡張機能は拡張機能の一覧には表示されません。
  • 学習機能つき迷惑メールフィルターを有効/無効にするオプションがすべてのアカウントに適用できるように見えます(Tools / Junk Mail Controls / Adaptive Filters)。しかしこれは アカウントごとの オプションです。特定のアカウントにこのオプションを設定するには、'Settings' パネルの 'Account:' ドロップダウンからそのアカウントを選択してから 'Adaptive Filters' パネルに移り、このオプションを設定します。そして必要なだけ各アカウントに対してこの作業を繰り返します。
  • デフォルトのテーマほぼ完成しつつありますが、まだいくつかのクラシックテーマが残っています。新しいテーマへの移行が済んでいないことに関するバグは報告しないでください。
