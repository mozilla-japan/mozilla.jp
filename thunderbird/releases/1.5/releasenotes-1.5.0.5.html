Thunderbird 1.5.0.5 の新機能と改良点

Thunderbird 1.5.0.5 は、ユーザーの皆さんにより安全なインターネット体験をお届けする私たちの取り組みの一環として提供される、安定性とセキュリティを向上させたアップデート版です。すべてのユーザーにこの最新版へのアップグレードを推奨します。

  • 安定性の向上
  • いくつかのセキュリティ問題の修正

注目すべきバグの修正のより詳細なリスト が The Rumbling Edge にあります。

リリース日: 2006/07/27

ダウンロードとインストール

システム動作環境

インストールする前にあなたのコンピューターが システム動作環境 を満たしているか確認してください。今回のバージョンより、Mac OS X では v10.2 以上が必要となります。

ダウンロード

Mozilla Foundation では Windows、Linux、Mac OS X 用の Thunderbird を提供しています。Thunderbird を入手するには 製品情報ページ を訪れてください。Mozilla Foundation が提供していない、他のシステム・言語向けのビルドについては、このページの最後にある「提供されたビルド」の項目をご覧ください。

インストール

Thunderbird のインストーラあるいは圧縮アーカイブをダウンロードしたら次の説明に従ってインストールしてください。

Windows

インストーラ Thunderbird Setup 1.5.0.5.exe をダブルクリックしてインストールを始めます。

Mac OS X

Thunderbird の圧縮ディスクイメージをダブルクリックしてマウントします。すでにブラウザーがこのディスクイメージを展開、マウントしているかもしれません。Thunderbird 1.5.0.5 ディスクイメージをダブルクリックして Thunderbird アプリケーションをハードディスクにドラッグします。Dock に追加したい場合は、Thunderbird のアイコンを Dock にドラッグしてください。

その他のプラットフォーム

圧縮されたアーカイブを展開し thunderbird を実行します。

アンインストール

Thunderbird をアンインストールするには次の手順に従ってください：

Windows

スタート メニューから コントロールパネル を選択します。コントロールパネルが表示されたら プログラムの追加と削除 をダブルクリックします。 リストの中の "Mozilla Thunderbird (1.5.0.5)" を見つけて 削除 をクリックしてアンインストールします。

Mac OS X

Thunderbird アプリケーションをゴミ箱にドラッグします。

その他のプラットフォーム

thunderbird ファルダを削除します。

これらの手順でアンインストールを完了しても、また次に Thunderbird をインストールする場合を考えて、あなたのプロファイルは削除されず、そのまま保存されます。Thunderbird のプロファイルを削除したい場合は、以下の「プロファイルフォルダー」の項目に書かれているフォルダーを削除してください。なお、一度プロファイルを削除してしまうと、ブックマークや保存されたパスワード、個人設定、その他のデータがすべで消去されてしまいますので、次に Thunderbird を使う場合は新しいプロファイルで起動することになります。

既知の問題

ここには、Thunderbird 1.5.0.5 に関する既知の問題の一部が載っています。新たにバグを報告する前にこのリストをご覧になり、今後も定期的にチェックするようにしてください (リリースに新しいバグが見つかった場合は更新されます)。

拡張機能とテーマ

このベータ版では、新しい Extension Compatibility and Update 機構が備わっていますが、これに関する問題に遭遇する可能性があります。私たちが認識している 3 つの問題について、それぞれ簡単な回避策があります。

  • Thunderbird 1.5.0.5 インストーラの実行後、Extension Compatibility and Update ウィザードが表示されない場合は、Extension Manager を開いて手動で更新を確認してください。
  • Extension Compatibility and Update ウィザードが表示されても完了しない場合は、何か他のウィンドウにフォーカスを移してから、再度ウィザードに戻ってください。これで問題が解決しない場合は、[Cancel] を選択し、Extension Manager を開いて手動で更新を確認してください。
  • Extension Compatibility and Update ウィザードが拡張機能の更新をダウンロード・インストールしても、それらの更新が適用されない場合は、Extension Manager を開いて手動で更新を確認してください。
  • Intel Mac では、PowerPC 専用のバイナリを含む拡張機能は、Intel ネイティブまたは Universal 版に更新するか、Rosetta を有効にしない限り、利用できません。Rosetta を有効にするには、Thunderbird のアイコンをクリックして [情報を見る] を選択し、[Rosetta を使って開く] にチェックを入れてください。

Thunderbird 1.0 にインストールされている拡張機能は Thunderbird 1.5.0.5 と互換性がなく、アップデートが必要な場合があります。何か問題があった場合は、その拡張機能の作者に報告してください。Thunderbird 1.5.0.5 をインストールする際、お使いの拡張機能やテーマは、Thunderbird 1.5.0.5 と互換性があること、または、互換性のある新しい拡張機能やテーマが提供されていることが確認されるまで、すべて無効化されます。これは、Thunderbird がバージョンごとに変更点を含んでおり、その変更された部分の機能を統合した一部の拡張機能やテーマは、Thunderbird の新バージョンで使うと問題を引き起こす可能性があるためです。お気に入りの拡張機能やテーマがアップデートされておらず、Thunderbird の新バージョンと互換性がない場合は、作者に連絡を取ってアップデートを促してください。

すべての OS

  • 一部のファイアウォールソフトが、Thunderbird の実行を静的にブロックしてしまう場合があります。たいていのファイアウォールは、プログラムによるインターネットへの接続を許可するかどうかを尋ねてきます。Thunderbird には常にアクセスを許可することが大切です。アクセスを拒否してしまった場合は、ファイアウォールの設定画面を開いて、アクセスを許可してください。ファイアウォールの中には、無効になっていてもアクセスが拒否されてしまうものもあります。特定のファイアウォールに関する情報はこちらをご覧ください。

  • マルチユーザーシステムで、Thunderbird をアクセス制限のある場所にインストールした場合、その場所にアクセス可能なユーザーがまず最初に Thunderbird を起動する必要があります。そうすることで、初回起動ファイルがすべて生成されます。この作業を行わないと、インストールされた場所に書き込み権限を持たないユーザーが Thunderbird を起動しようとした場合、Thunderbird が必要とする初回起動ファイルが正しく生成されません。

  • デフォルトのスペルチェック辞書は英語のみとなっています。英語以外の辞書を追加するには インストール方法 を参照してください。

  • 共通受信トレイと POP3 アカウント用受信トレイの使用を切り替える場合は、必ずメッセージを他のフォルダーにコピーしておいてください。

  • ソフトウェアアップデートは、Thunderbird がインストールされている場所に書き込み権限のないユーザーが実行すると失敗します。これは、ソフトウェアアップデートがその場所にあるファイルを置き換え・作成する必要があるためです。

Windows

  • 共有マシンで、アクセス権の限られたユーザーが自分の書き込み権限のある場所に Thunderbird をインストールした場合でも、マイナスの副作用が発生する場合があります (既定のメールクライアントその他のキーが正しく設定されないなど)。それでも Thunderbird は使えます。アクセス権の限られたユーザーがインストールする場合、アクセス制限のある場所や共有の場所にインストールしてはいけません。インストールに失敗する可能性があります。

  • Windows ユーザーは、Thunderbird 向け Palm Sync 拡張機能 をインストールすることで、Palm デバイスとアドレス帳を同期させることができます。ファイルをローカルに保存して、Thunderbird の拡張機能ダイアログからインストールしてください。この拡張機能は拡張機能一覧には表示されません。拡張機能を使うには、インストール後 Thunderbird を再起動する必要があります。

Mac OS X

  • Intel Core プロセッサー搭載した Macintosh コンピューターで、Rosetta 上で Thunderbird 1.5 または 1.5.0.2 をお使いの場合、自動アップデートシステムを通じて PowerPC 専用の Thunderbird 1.5.0.5 に更新されてしまいます。Intel Mac 向けのネイティブサポートを受けるには、製品情報ページから Universal 版の Thunderbird 1.5.0.5 をダウンロードしてください。
  • Intel Mac では、ネイティブの場合も、Rosetta 上で実行している場合も、Talkback 機能は利用できません。アプリケーションがクラッシュした際には、Apple のクラッシュ報告プログラムが起動します。

Linux/Unix システム

  • GNOME との統合は Fedora Core 3 では正しく機能しません。Fedora Core 3 ユーザーは、linc-1.0.3-3.1.i386.rpm をダウンロード・インストールする必要があります。この RPM をインストールしたら、Thunderbird をインストールしたディレクトリーで次のコマンドを実行してください (書き込み権限が必要です)。

    touch .autoreg

    次に Thunderbird を起動したときから GNOME 統合が有効になります。

よくある質問

どのような協力ができますか?
私たちは少しでも多くの配布経路を必要としています。できる限り多くの友人や家族、同僚を Thunderbird に乗り換えさせることがあなたの使命です。もしあなたが学生なら、学校で Thunderbird を配布してください。リリースについて Slashdot [日本語版] やその他のニュースサイトに記事を投稿してください。あなたのブログから情報を流してください。例えばあの Kazaa のように、インターネットを通じた大規模な配布が可能です。
なぜ私が送ったメールに返事をくれないのですか?
フォーラム をご利用ください。Thunderbird チームは常時このフォーラムをチェックしています。私たちは毎日多くのメールを受け取っているので、あなたが送ったメールは見過ごされる可能性もあります。
テーマとアドオン (拡張機能) はどこで手に入りますか?
テーマと拡張機能は Mozilla Update からダウンロードできます。
誰が Thunderbird の作業をしているのですか?
非常に多くの人が関わっています。Thunderbird に貢献している人たちの一覧を見るには、[ヘルプ] メニューの [Mozilla Thunderbird について] に書かれているクレジットをご覧ください。
