Firefox リリースノート

Preview Release (Greenlane)

Firefox は、ウェブブラウジングをかつてなく効率的にする、フル機能の高速ブラウザーです。

Firefox Preview Release (以下、PR) は テクノロジープレビュー です。このソフトウェアは、ほとんどの場合、デフォルトブラウザーとして信頼できるほど十分に機能しますが、そのパフォーマンスや安定性については一切保証できません。これはまだ (バージョンが 1.0 に達していない) リリース前の製品ですので、基幹システムとして信頼すべきではありません。詳しくは ライセンス規約 をご覧ください。

このリリースノートには、Firefox PR に関する 新機能、ダウンロード、インストール方法、既知の問題、よくある質問 などが載っています。Bugzilla にバグを報告する前に、このリリースノートと バグの登録方法 をよくお読みください。

Firefox に関するあなたのフィードバックをお待ちしています。MozillaZine によって運営されている Firefox フォーラム にぜひ参加してください。

新機能

今回のリリースの新機能をご紹介します。

ライブブックマーク
RSS フィードをブックマーク内で購読し、参照できるようになりました。<link> タグを使って RSS フィードを配信しているページを訪れると、ステータスバーに アイコンが表示されます。これをクリックすると、提供されているフィードの一覧を見ることができます。その中から購読したいものを選択すると、フィードから取得した最近の投稿をすべて含んだブックマークフォルダーが追加されます。
検索機能の向上
新しい検索ツールバーが実装され、検索機能がより使いやすく、パワフルになりました。検索ツールバー (ブラウザーウィンドウの下部に表示されます) は、ユーザーが入力したページ上のテキストを自動的に発見するなど、便利なハイライト機能が付属しています。
迷惑な仕掛けの管理とセキュリティ保護
一度ブロックしたポップアップを開いて確かめることが可能になりました。また、addons.mozilla.org 以外のサイトからソフトウェアをインストールしようとすると、拡張機能インストールシステムによって完全にブロックされるようになりました。ユーザーは、他のサイトをホワイトリストに追加して、ソフトウェアのインストールを許可することができますが、ソフトウェアが自動的にインストールされてしまうことは 絶対に ありません。これらの措置に加えて、フィッシング攻撃を防いだり、安全な接続を通じてページをブラウズしているときにそれを分かりやすく表示するといった対策がいくつか講じられています。
ブックマークの改良
より確かなサイトアイコンの表示、ブックマークウィンドウの分割ビューなど、ブックマーク機能にも数多くの改良が加えられています。
パスワードのより強力な暗号化を実現
パスワードマネージャーで保存されたパスワードが、「マスターパスワード」を作成することによって、強力な暗号化アルゴリズムを使って、より簡単に暗号化できるようになりました。マスターパスワードを作成すると、セッションごとにマスターパスワードを入力するよう求められます。ここで入力しないと、パスワードマネージャーはサイトのログインに必要な情報 (ユーザー名とパスワード) をフォームに自動入力しません。コンピューターを他の人と共有していて、セキュリティを向上させたい場合には特に便利な機能です。
Internet Explorer との互換性向上
サイト互換性のため、判別を含まない document.all サポートが追加されました。また、キーボードアクセラレータの互換性を改善したことで、Internet Explorer ユーザーがよりスムーズに移行できるようになりました。
その他数多くのバグの修正

詳しくは The Burning Edge: Bigger Picture をご覧ください。

インストール

Firefox のインストーラか圧縮アーカイブをダウンロードしたら、以下の手順に従ってインストールしてください。

Windows

インストーラ (Firefox Setup 1.0PR.exe) をダブルクリックするとインストールが開始されます。

Mac OS X

Firefox の圧縮ディスクイメージをダブルクリックしてマウントします。お使いのブラウザーによっては、イメージが自動的に展開・マウントされる場合もあります。マウントされた Firefox アプリケーションをハードディスクにドラッグしてください。Dock に追加したい場合は、アイコンを Dock にドラッグしてください。

Linux/GTK2

tarball を展開して、以下のようにインストーラを実行してください。

tar -xzvf firefox-1.0PR-i686-linux-gtk2+xft.tar.gz
cd firefox-0.10-i686-linux-gtk2+xft/
./firefox-installer

その他のプラットフォーム

圧縮アーカイブを展開して firefox を実行してください。

アンインストール

Firefox をアンインストールするには、以下の手順に従ってください。

Windows

[スタート] メニューから [コントロール パネル] を選択してください。パネルが開いたら [プログラムの追加と削除] をダブルクリックしてください。リストの中から "Mozilla Firefox" を見つけて [削除] ボタンをクリックするとアンインストールが開始されます。

Mac OS X

Firefox アプリケーションをゴミ箱にドラッグしてください。

その他のプラットフォーム

firefox フォルダーを削除してください。

これらの手順でアンインストールを完了しても、また次に Firefox をインストールする場合を考えて、あなたのプロファイルは削除されず、そのまま保存されます。Firefox の プロファイルフォルダー を削除したい場合は、以下の手順に従ってください。

Windows

Application Data フォルダーを探します。このフォルダーは通常 Documents and Settings¥<ユーザー名>¥Application Data にあります。Application Data フォルダーは隠しフォルダーなので、Windows エクスプローラでこれらを表示させるよう設定してください。ユーザー名が "Joe" で、Documents and Settings フォルダーが C ドライブにある場合、あなたのプロファイルフォルダーは C:¥Documents and Settings¥Joe¥Application Data¥Mozilla¥Firefox にあります。この Firefox フォルダーを削除すればプロファイルデータが削除されます。

Mac OS X

自分のホームフォルダーの中にあるライブラリフォルダーを開き、Firefox フォルダー (~/Library/Firefox/) を削除してください。

その他のプラットフォーム

~/.firefox を削除してください。

Firefox 0.8 からの注意すべき変更点

プロファイルフォルダー

Firefox は、以下の場所にユーザーデータを保存するようになりました。

WindowsDocuments and Settings¥<UserName>¥Application Data¥Mozilla¥Firefox
Mac OS X~/Library/Application Support/Firefox
Linux~/.mozilla/firefox/

Firefox PR にアップグレードする前に Firefox 0.8 をデフォルトブラウザーとして使っていた場合、プロファイルのデータは新しい場所にコピーされます。古い "Phoenix" フォルダーは後で時間があるときにでも削除してください。

旧バージョンの拡張機能とテーマ

("No XBL Binding for Browser" というエラーが出てブラウザーが起動しない、あるいはテーマが壊れるといった) Firefox 0.8 に見つかったいくつかの問題を防ぐため、PR に移行する際、拡張機能はすべて自動的に無効化されます。これまで 0.9 を使っていた場合は、PR を初めて起動した際、addons.mozilla.org に接続し、新しい互換性のある拡張機能を探します。もし見つかれば、ダウンロード、インストールします。0.8 を使っていた場合は、新しい互換性のあるバージョンをご自身で探してください。

既知の問題

ここには、Firefox PR に関する既知の問題の一部が載っています。新たにバグを報告する前にこのリストをご覧になり、今後も定期的にチェックするようにしてください (リリースに新しいバグが見つかった場合は更新されます)。

  • Firefox PR をアクセス権限の限られた場所にインストールした場合、その場所にアクセスできるユーザーがまず最初に Firefox を実行する必要があります。そうすることで、初回起動ファイルがすべて生成されます。アクセス制限のあるユーザーが最初に Firefox を起動してしまうと、それらのファイルが正しく生成されず、アプリケーションが無限再起動ループに突入してしまうという問題があります。この問題は今後のリリースで修正されます。また、Mac OS X ユーザーの方は、Firefox をディスクイメージから実行しないでください。万が一実行してしまうと、無限再起動ループが起こってしまいます。Firefox の無限ループを止めるには、ターミナルを開いて "killall firefox-bin" と入力してください。
  • Mac OS X で Firefox のブラウザーウィンドウが表示されない場合、一度 Cmd+Q で Firefox を終了し、~/Library/Application Support/Firefox/Profiles/default.abc/ (abc はランダムな文字列) を開いて localstore.rdf を削除してください。それから Firefox を再起動してください。このとき、ツールバーのカスタマイズ内容やウィンドウの位置など一部の設定が失われます。
  • Firefox は Mozilla Application Suite から多くのコードを利用しています。Firefox に含まれる問題の多くは、実際には Mozilla のコアコードにおける問題です。ページのコンテンツや接続性に関する問題を発見した場合、それはおそらく Mozilla の問題ですから、Bugzilla では Firefox プロダクトではなく Browser プロダクトに対して報告してください。
  • 私たちはまだ、この新しい自動拡張機能アップデートの仕組みを簡単にしようと考えており、できる限り早く、残っている問題を取り除こうと試みています。拡張機能とテーマの作者は、Firefox の新しいバージョンがリリースされるたびに、addons.mozilla.org (あるいは独自のカスタムアップデートサービス) のリストを更新する必要があります。そうすることで、それらの拡張機能やテーマの互換性を知ることができます。もし、PR にアップグレードしたことで、お気に入りの拡張機能やテーマが無効になってしまった場合は、それらの作者に連絡を取って、リストやカスタムアップデートサービスのファイルを更新するよう働きかけてください。それらが addons.mozilla.org に登録されていない場合は、登録手続きを行うよう作者に促してください。それによって、バージョンの変更をとても簡単に処理できます。

Firefox に関するその他の問題、よくある質問、便利な使い方、一般的なヘルプについては、David Tenser の大変便利なヘルプサイト Firefox Help や、MozillaZine によって運営されている Firefox フォーラム をチェックしてください。

トラブルシューティング

前のバージョンの Firefox からアップグレードする際、あらかじめ すべての拡張機能を無効に しておけば、新たにプロファイルを作成する必要はありません。拡張機能を無効にするには、前のバージョンの Firefox を開いて、Tools (ツール) メニューの Options (オプション) を選択し、Extensions (拡張機能) パネルをクリックします。リストからそれぞれの拡張機能をクリックし、Disable Extension (拡張機能を無効にする) を選択してください。すべて無効にしたら OK をクリックして Options ウィンドウを閉じてください。これで安心して新しいバージョンの Firefox をインストールできます。

Firefox をインストールしたら、前にインストールしていた拡張機能が Firefox の新バージョンに対応しているかどうか確認してから、再度インストールしてください。今後この手順は自動化される予定です。

前のバージョンの Firebird や Phoenix で拡張機能を有効にしたまま同じプロファイルを使用して問題が起きた場合は、前のバージョンのブラウザーを起動し、拡張機能を無効にしてから、再度新しいバージョンの Firefox を起動してください。

ブックマーク、ダウンロード、ウィンドウの配置、ツールバー、履歴その他の設定に関して不可解な問題に遭遇した場合は、バグを報告する前に新しいプロファイルを作成して問題が再現するかどうか確かめてください。プロファイルを新規作成するには、コマンドライン引数 -P を付けて Firefox を起動し、Manage Profiles (プロファイルを管理) ボタンをクリックして Create Profile (プロファイルを作成) を選択してください。設定ファイル (ブックマーク、保存したパスワードなど) をひとつずつ手作業で移行し、そのたびに問題が再現するかどうか確認してください。特定のプロファイルデータが問題を起こすことが分かった場合は、バグを登録してそのファイルを添付してください。

よくある質問

どのような協力ができますか?
私たちは少しでも多くの配布経路を必要としています。Spread Firefox に参加して、史上最大の草の根マーケティングキャンペーンの一員となりましょう。できる限り多くの友人や家族、同僚を Firefox に乗り換えさせることがあなたの使命です。もしあなたが学生なら、学校で Firefox を配布してください。リリースについて Slashdot [日本語版] やその他のニュースサイトに記事を投稿してください。あなたの blog から情報を流してください。例えばあの Kazaa のように、インターネットを通じた大規模な配布が可能です。Firefox の話題をどんどん広めてください!
私は別のブラウザーを使っています。なぜなら...
もしあなたが Firefox を使っていないのなら、開発チームにその理由を教えてください。私たちは Firefox フォーラム に寄せられたフィードバックに目を通しています。
なぜ私が送ったメールに返事をくれないのですか?
フォーラム をご利用ください。Firefox チームは常時このフォーラムをチェックしています。私たちは毎日多くのメールを受け取っているので、あなたが送ったメールは見過ごされる可能性もあります。
テーマとアドオン (拡張機能) はどこで手に入りますか?
テーマと拡張機能は Mozilla Update からダウンロードできます。
誰が Firefox の作業をしているのですか?
非常に多くの人が関わっています。Firefox に貢献している人たちの一覧を見るには、Help (ヘルプ) メニューの About Mozilla Firefox (Mozilla Firefox について) に書かれているクレジットをご覧ください。
Firefox のソースコードはどこにありますか?
cvs.mozilla.org にあります。Mozilla トランクと mozilla/browser と mozilla/toolkit を組み合わせたものになります。
Phoenix Firebird の名称に何が起きたのですか?
商標の問題から名称を変更する必要がありました。新名称は Firefox です。詳しくは プレスリリース と よくある質問 をご覧ください。
メールクライアントはどこにあるのですか?
Firefox は、お使いのシステムでどのアプリケーションがデフォルトのメールクライアントになっていても、それと併せて使うことができますが、Firefox ブラウザーにぴったりな、Mozilla の次世代メールクライアント Mozilla Thunderbird を使ってみることをお勧めします。
