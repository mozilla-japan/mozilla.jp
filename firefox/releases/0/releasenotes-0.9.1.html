Firefox リリースノート

バージョン 0.9.1 (One Tree Hill)

Firefox の新バージョンがリリースされています。最新版に関する新着情報とダウンロードは こちらから どうぞ。

Firefox は、ウェブブラウジングをかつてなく効率的にする、フル機能の高速ブラウザーです。Firefox に関する 詳しい情報 もご覧ください。

Firefox 0.9.1 は テクノロジープレビュー です。このソフトウェアは、ほとんどの場合、デフォルトブラウザーとして信頼できるほど十分に機能しますが、そのパフォーマンスや安定性については一切保証できません。これはまだ (バージョンが 1.0 に達していない) リリース前の製品ですので、基幹システムとして信頼すべきではありません。詳しくは ライセンス規約 をご覧ください。

このリリースノートには、Firefox 0.9.1 のリリースに関する 新機能、ダウンロード、インストール方法、既知の問題、よくある質問 などが載っています。Bugzilla にバグを報告する前に、このリリースノートと バグの登録方法 をよくお読みください。

Firefox に関するあなたのフィードバックをお待ちしています。MozillaZine によって運営されている Firefox フォーラム にぜひ参加してください。

新機能

今回のリリースの新機能をご紹介します。

新しいデフォルトテーマ
デフォルトテーマがアップデートされ、3 種類すべてのプラットフォームにおいて一貫したアピアランスが提供されるようになりました。Firefox 0.9.1 の Windows 版には、すっきりと引き締まった新テーマが統合されています。今後のリリースで Linux 版 (GNOME) テーマの微調整も行われます。
包括的なデータ移行支援
Firefox への乗り換えがかつてなく簡単になりました。Firefox は、お気に入り、履歴、個人設定、Cookie、パスワードなど、様々なデータを Internet Explorer からインポートできるようになりました。また、Mozilla 1.x、Netscape 4/6/7、Opera からのインポートも可能です。今後のリリースで、Mac OS X 版と Linux 版にも、Safari、OmniWeb、Konqueror といったブラウザーからの移行支援機能が実装されます。
拡張機能・テーママネージャー
拡張機能マネージャーとテーママネージャーが新たに加わり、アドオンの管理とアップデートが大変便利になりました。また、Firefox の新バージョンがリリースされるとアップデートを通知してくれる「スマートアップデート機能」も搭載されています。
ダウンロードサイズの削減
Firefox の Windows 版インストーラが、わずか 4.7 MB になりました。
ヘルプ
新しいオンラインヘルプシステムが利用可能になりました。
数多くのバグの修正と機能向上
画像のコピー、オートコンプリート (自動補完) リストから個々のアイテムを削除できる機能、gnome-vfs による GNOME 上での SMB/SFTP サポート、ブックマークや検索機能の改良、その他様々な洗練が、ブラウジング経験をより一層豊かなものにします。

Linux/GTK2 ユーザー向け

インストーラ
Firefox の Linux/GTK2 版はインストーラで提供されるようになりました。この新しいインストーラによって、インストールの手順がとても簡単になります。
ルック＆フィールのアップデート
現在進められている改良によって、Firefox のメニューなどが GTK2 テーマにより忠実になります。
Linux/GTK2 版 Talkback
このクラッシュ報告ツールを利用して Talkback レポートを送信し、クラッシュの原因突き止めにご協力ください。

詳しくは The Burning Edge: Bigger Picture をご覧ください。

インストール

Firefox のインストーラか圧縮アーカイブをダウンロードしたら、以下の手順に従ってインストールしてください。

最初に、これらのプレビューリリースについては、まず前のバージョンの Firefox をアンインストールすることを強くお勧めします。古いバージョンの上に重ねてインストールすると、予測不可能な問題が起きる場合があります。古いバージョンに上書きインストールして、バグを登録しようと思っている場合は、その前に、何も入っていない空のディレクトリーにクリーンインストールを実行してください。

Windows

インストーラ (FirefoxSetup-0.9.1.exe) をダブルクリックするとインストールが開始されます。

注意: 別の (以前のバージョンの) Firefox に上書きインストールしてはいけません。Firefox 0.9.1 を Firefox 0.8 と同じフォルダーにインストールしたい場合は、先に Firefox 0.8 をアンインストールしてください。このアップグレードの問題は今後のリリースで修正されます。

Mac OS X

Firefox の圧縮ディスクイメージをダブルクリックしてマウントします。お使いのブラウザーによっては、イメージが自動的に展開・マウントされる場合もあります。マウントされた Firefox アプリケーションをハードディスクにドラッグしてください。Dock に追加したい場合は、アイコンを Dock にドラッグしてください。

Linux/GTK2

tarball を展開して、以下のようにインストーラを実行してください。

tar -xzvf firefox-0.9.1-i686-linux-gtk2+xft.tar.gz
cd firefox-0.9.1-i686-linux-gtk2+xft/
./firefox-installer

その他のプラットフォーム

圧縮アーカイブを展開して firefox を実行してください。

アンインストール

Firefox をアンインストールするには、以下の手順に従ってください。

Windows

[スタート] メニューから [コントロール パネル] を選択してください。パネルが開いたら [プログラムの追加と削除] をダブルクリックしてください。リストの中から "Mozilla Firefox (0.9.1.)" を見つけて [削除] ボタンをクリックするとアンインストールされます。

Mac OS X

Firefox アプリケーションをゴミ箱にドラッグしてください。

その他のプラットフォーム

firefox フォルダーを削除してください。

この手順でアンインストールを完了しても、また次に Firefox をインストールする場合を考えて、あなたのプロファイルは削除されず、そのまま保存されます。Firefox の プロファイルフォルダー を削除したい場合は、以下の手順に従ってください。

Windows

Application Data フォルダーを探します。このフォルダーは通常 Documents and Settings¥<ユーザー名>¥Application Data にあります。Application Data フォルダーは隠しフォルダーなので、Windows エクスプローラでこれらを表示させるよう設定してください。ユーザー名が "Joe" で、Documents and Settings フォルダーが C ドライブにある場合、あなたのプロファイルフォルダーは C:¥Documents and Settings¥Joe¥Application Data¥Mozilla¥Firefox にあります。この Firefox フォルダーを削除すればプロファイルデータが削除されます。

Mac OS X

自分のホームフォルダーの中にあるライブラリフォルダーを開き、Firefox フォルダー (~/Library/Firefox/) を削除してください。

その他のプラットフォーム

~/.firefox を削除してください。

Firefox 0.8 からの注意すべき変更点

プロファイルフォルダー

Firefox 0.9.1 は、以下の場所にユーザーデータを保存するようになりました。

WindowsDocuments and Settings¥<UserName>¥Application Data¥Mozilla¥Firefox
Mac OS X~/Library/Application Support/Firefox
Linux~/.mozilla/firefox/

Firefox 0.9.1 にアップグレードする前に Firefox 0.8 をデフォルトブラウザーとして使っていた場合、プロファイルのデータは新しい場所にコピーされます。古い "Phoenix" フォルダーは後で時間があるときにでも削除してください。

ナイトリービルドユーザーへの注意

Firefox 0.8 から 0.9.1 の間に、プロファイルの場所が Application Data¥Firefox (Windows)、~/Library/Firefox (Mac OS X) または ~/.firefox (Linux) から上記の場所に変更されました。0.9.1 を初めて起動する際、プロファイルに関する情報は新しい場所にコピーされますが、データは移動されません。Application Data¥Firefox、~/Library/Firefox または ~/.firefox フォルダーを削除してはいけません。このフォルダーにはまだプロファイルが残っており、新しい場所にある profiles.ini ファイルはそれらのプロファイルを参照するようになっているからです。手作業でファイルを移動し、profile.ini を編集して、レジストリが新しい場所を指すようにアップデートする必要があります。Firefox 0.8 ユーザーはこれについて心配する必要はありません。この内容は、コミュニティのテスターと、最新のナイトリービルドを追いかけている人たちだけに当てはまるものです。

旧バージョンの拡張機能とテーマ

("No XBL Binding for Browser" というエラーが出てブラウザーが起動しない、あるいはテーマが壊れるといった) Firefox 0.8 に見つかったいくつかの問題を防ぐため、初めて 0.9.1 を起動する際、拡張機能はすべて自動的に無効化されます。また、テーマもデフォルトにリセットされます。拡張機能の API が変更されたため、これまでインストールしていた拡張機能については、Firefox 0.9.1 と互換性のある新バージョンを探してください。Firefox 0.9.1 以降、これらのアップデート作業はよりシームレスになるはずです。

既知の問題

ここには、Firefox 0.9.1 に関する既知の問題の一部が載っています。新たにバグを報告する前にこのリストをお読みになり、今後も定期的にチェックするようにしてください (リリースに新しいバグが見つかった場合は更新されます)。

  • Firefox 0.9.1 をアクセス権限の限られた場所にインストールした場合、その場所にアクセスできるユーザーがまず最初に Firefox を実行する必要があります。そうすることで、初回起動ファイルがすべて生成されます。アクセス制限のあるユーザーが最初に Firefox を起動してしまうと、それらのファイルが正しく生成されず、アプリケーションが無限再起動ループに突入してしまうという問題があります。この問題は今後のリリースで修正されます。
  • Mac OS X で Firefox のブラウザーウィンドウが表示されない場合、一度 Cmd+Q で Firefox を終了し、~/Library/Application Support/Firefox/Profiles/default.abc/ (abc はランダムな文字列) を開いて localstore.rdf を削除してください。それから Firefox を再起動してください。このとき、ツールバーのカスタマイズ内容やウィンドウの位置など一部の設定が失われます。
  • あるサイトが、他のサイトによって生成された frameset に対してリンクを向けたりフォームを送信することに関して、セキュリティの問題を修正するため、いくつかの変更を行いました。しかし、これらの変更によって、一部のサイトで問題が起きる可能性があります。このフレーム機能を正当に利用しているサイトで問題に遭遇し、今まで通りに使いたい場合は、about:config を使って docshell.frameloadcheck.disabled の値を true に設定すれば、セキュリティチェックを無効化できます。もしこれでも問題が起きるサイトがあれば、バグを登録して、そのサイトの URL を教えてください。
  • Firefox は Mozilla Application Suite から多くのコードを利用しています。Firefox に含まれる問題の多くは、実際には Mozilla のコアコードにおける問題です。ページのコンテンツや接続性に関する問題を発見した場合、それはおそらく Mozilla の問題ですから、Bugzilla では Firefox プロダクトではなく Browser プロダクトに対して報告してください。
  • Windows 版では、メニューやブックマークツールバー上のフォルダーが正しく折り畳まれないことがあります。
  • テーマインストールの [Use this theme] (このテーマを使用する) チェックボックスは正しく機能しませんので、Tools (ツール) メニューから Themes (テーマ) を選択したあと、使いたいテーマを選択し、[Use Theme] をクリックして切り替える必要があります。また、一部のテーマは再起動しないと完全に切り替わらないことがあります。
  • リリース候補 (RC) を含めて、バージョン 0.8 から 0.9.1 までの間のナイトリービルドを使っている方で、もし Firefox が起動時に無限再起動ループに突入した場合は、プロファイルフォルダーの中にある extensions フォルダーを削除 するようお勧めします。
  • 一部のユーザーから、ダウンロードウィンドウのパフォーマンスに問題があるとの報告が寄せられています。ダウンロードウィンドウの表示が遅いと感じられる場合は このパッチ をインストールしてみてください。
  • 拡張機能アップデートとソフトウェアアップデートはまだ十分にテストされていません。このため、いくつかの問題が生じる可能性があります。これらの問題は 1.0 Beta で解決されます。0.9.1 リリースの時点では、まだ update.mozilla.org にアップデートサービスが用意されていないため、アップデートは利用できません。利用可能なアップデートのステータス通知を見ても、アップデートは「ありません」と表示されるはずです。

Firefox に関するその他の問題、よくある質問、便利な使い方、一般的なヘルプについては、David Tenser の大変便利なヘルプサイト Firefox Help [日本語版] や、MozillaZine によって運営されている Firefox フォーラム をチェックしてください。

トラブルシューティング

前のバージョンの Firefox からアップグレードする際、あらかじめ すべての拡張機能を無効に しておけば、新たにプロファイルを作成する必要はありません。拡張機能を無効にするには、前のバージョンの Firefox を開いて、Tools (ツール) メニューの Options (オプション) を選択し、Extensions (拡張機能) パネルをクリックします。リストからそれぞれの拡張機能をクリックし、Disable Extension (拡張機能を無効にする) を選択してください。すべて無効にしたら OK をクリックして Options ウィンドウを閉じてください。これで安心して新しいバージョンの Firefox をインストールできます。

Firefox をインストールしたら、前にインストールしていた拡張機能が Firefox の新バージョンに対応しているかどうか確認してから、再度インストールしてください。今後この手順は自動化される予定です。

前のバージョンの Firebird や Phoenix で拡張機能を有効にしたまま同じプロファイルを使用して問題が起きた場合は、前のバージョンのブラウザーを起動し、拡張機能を無効にしてから、再度新しいバージョンの Firefox を起動してください。

ブックマーク、ダウンロード、ウィンドウの配置、ツールバー、履歴その他の設定に関して不可解な問題に遭遇した場合は、バグを報告する前に新しいプロファイルを作成して問題が再現するかどうか確かめてください。プロファイルを新規作成するには、コマンドライン引数 -P を付けて Firefox を起動し、Manage Profiles (プロファイルを管理) ボタンをクリックして Create Profile (プロファイルを作成) を選択してください。設定ファイル (ブックマーク、保存したパスワードなど) をひとつずつ手作業で移行し、そのたびに問題が再現するかどうか確認してください。特定のプロファイルデータが問題を起こすことが分かった場合は、バグを登録してそのファイルを添付してください。

よくある質問

どのような協力ができますか?
私たちは少しでも多くの配布経路を必要としています。できる限り多くの友人や家族、同僚を Firefox に乗り換えさせることがあなたの使命です。もしあなたが学生なら、学校で Firefox を配布してください。リリースについて Slashdot [日本語版] やその他のニュースサイトに記事を投稿してください。あなたの blog から情報を流してください。例えばあの Kazaa のように、インターネットを通じた大規模な配布が可能です。Firefox の話題をどんどん広めてください!
私は別のブラウザーを使っています。なぜなら...
もしあなたが Firefox を使っていないのなら、開発チームにその理由を教えてください。私たちは Firefox フォーラム に寄せられたフィードバックに目を通しています。
なぜ私が送ったメールに返事をくれないのですか?
フォーラム をご利用ください。Firefox チームは常時このフォーラムをチェックしています。私たちは毎日多くのメールを受け取っているので、あなたが送ったメールは見過ごされる可能性もあります。
テーマとアドオン (拡張機能) はどこで手に入りますか?
テーマと拡張機能は Mozilla Update からダウンロードできます。
誰が Firefox の作業をしているのですか?
現在の開発スタッフは、Ben Goodger (Mozilla Foundation)、Brian Ryner (IBM)、Pierre Chanial、Blake Ross、Dave Hyatt、Benjamin Smedberg、Darin Fisher、そして Mozilla のコードベースに貢献している多数のコミュニティのメンバーです。Asa Dotzler、Leaf Nunes と QA コンタクトチームが、バグの選別、リリース、その他諸々の作業を処理しています。詳しくは Help (ヘルプ) メニューの About Mozilla Firefox (Mozilla Firefox について) に書かれているクレジットをご覧ください。
Firefox のソースコードはどこにありますか?
cvs.mozilla.org にあります。Mozilla トランクと mozilla/browser と mozilla/toolkit を組み合わせたものになります。
Firefox は、ユーザーインターフェイスにちょっとだけ手を加えた、ただの Mozilla では?
Firefox には数々の独自機能や数え切れないほどの改良を加えており、まったくの別物です。プロジェクト開始以来、すでに 12 万行以上のコードが CVS ディレクトリーに追加、変更されています。
Phoenix Firebird の名称に何が起きたのですか?
商標の問題から名称を変更する必要がありました。新名称は Firefox です。詳しくは プレスリリース と よくある質問 をご覧ください。
メールクライアントはどこにあるのですか?
Firefox は、お使いのシステムでどのアプリケーションがデフォルトのメールクライアントになっていても、それと併せて使うことができますが、Firefox ブラウザーにぴったりな、Mozilla の次世代メールクライアント Mozilla Thunderbird を使ってみることをお勧めします。
