Firefox リリースノート

バージョン 0.8 (Royal Oak)

Firefox の新バージョンがリリースされています。最新版に関する新着情報とダウンロードは こちらから どうぞ。

Firefox は、ウェブブラウジングをかつてなく効率的にする、フル機能の高速ブラウザーです。Firefox に関する 詳しい情報 もご覧ください。

Firefox 0.8 は テクノロジープレビュー です。このソフトウェアは、ほとんどの場合、デフォルトブラウザーとして信頼できるほど十分に機能しますが、そのパフォーマンスや安定性については一切保証できません。これはまだ (バージョンが 1.0 に達していない) リリース前の製品ですので、基幹システムとして信頼すべきではありません。詳しくは ライセンス規約 をご覧ください。

このリリースノートには、Firefox 0.8 のリリースに関する 新機能、ダウンロード、インストール方法、既知の問題、よくある質問 などが載っています。Bugzilla にバグを報告する前に、このリリースノートと バグの登録方法 をよくお読みください。

Firefox に関するあなたのフィードバックをお待ちしています。MozillaZine [日本語版] によって運営されている Firefox フォーラム にぜひ参加してください。

新機能

今回のリリースの新機能をご紹介します。

  • Windows 版インストーラ

    Firefox の Windows 版はインストーラで提供されるようになりました。このインストーラは、インストール手順をよりスムーズにするだけでなく、一部のプラグインに関するよくある問題を解決します。

  • ダウンロードマネージャー

    合理化された新しいダウンロードマネージャーは、ユーザーが進めている作業を邪魔することなくダウンロードを整理します。ダウンロードマネージャーを表示させたり、設定を変更するには、Tools (ツール) メニューから Downloads (ダウンロード) を選択するか、Ctrl+E を押してください。

  • 新しいブックマーク追加ダイアログ

    ブックマーク追加ダイアログが強化され、ブックマークフォルダーの新規作成が可能になりました。

  • オフライン作業

    インターネットに接続しなくても、前に見たウェブページを表示できるようになりました。

  • ファイルタイプ処理の改良

    サーバーに誤って text/plain として送信されたバイナリファイル (.wma や .rar など) が、ブラウザー内にゴミのような文字のかたまりとして表示されることなく、適切に処理されるようになりました。

  • 新しい XPInstall フロントエンド

    テーマや拡張機能をインストールする際に表示されるダイアログが再設計され、より使いやすくなりました。

  • 新しい Mac 用デフォルトテーマ

    Mac OS X では、Windows や Linux で使われている Qute テーマの代わりに、ネイティブのユーザーインターフェイスに合わせた Pinstripe テーマが統合されました。

  • その他数多くのバグの修正

    フォームのオートコンプリート (自動補完) の無効化が機能するようになりました。また、タブを切り替える際に、アドレスバーに入力途中のテキストが記憶されるようになりました。その他にも数多くのバグの修正が行われています。

インストール

Firefox のインストーラか圧縮アーカイブをダウンロードしたら、以下の手順に従ってインストールしてください。

まず、以前のバージョンの Firebird や Phoenix を起動している場合は、Tools (ツール) メニューの Options (オプション) から Extensions (拡張機能) パネルを開き、すべての拡張機能を無効に してください。それができない場合は、Firefox 0.8 にアップグレードする前に必ず 新しいプロファイルを作成 してください。そうしないと、ブラウザーが正しく機能せず、"Cannot create XBL Binding for Browser" あるいは "Cannot create Browser Instance" といったエラーメッセージが表示されてしまいます。拡張機能の新バージョンを探す場合は、必ず Firefox 0.8 と互換性がある ことがはっきりと書かれているものをインストールしてください。

Windows

インストーラをダブルクリックするとインストールが開始されます。

注意: Custom Install を選択し、他のファイルが含まれるフォルダーをインストール先に指定して Clean Install を実行すると、それらのファイル���インストーラによって すべて削除 されてしまいます。このため、デフォルトの場所にインストールするか、新しいフォルダーを作成することを強くお勧めします。

Mac OS X

Firefox の圧縮ディスクイメージをダブルクリックしてマウントします。お使いのブラウザーによっては、イメージが自動的に展開・マウントされる場合もあります。マウントされた Firefox アプリケーションをハードディスクにドラッグしてください。Dock に追加したい場合は、アイコンを Dock にドラッグしてください。

その他のプラットフォーム

圧縮アーカイブを展開して firefox を実行してください。

アンインストール

Firefox をアンインストールするには、以下の手順に従ってください。

Windows

[スタート] メニューから [コントロール パネル] を選択してください。パネルが開いたら [プログラムの追加と削除] をダブルクリックしてください。リストの中から "Mozilla Firefox (0.8.)" を見つけて [削除] をクリックするとアンインストールされます。

Mac OS X

Firefox アプリケーションをゴミ箱にドラッグしてください。

その他のプラットフォーム

firefox フォルダーを削除してください。

この手順でアンインストールを完了しても、また次に Firefox をインストールする場合を考えて、あなたのプロファイルは削除されず、そのまま保存されます。Firefox の プロファイルフォルダー を削除したい場合は、以下の手順に従ってください。

Windows

Application Data フォルダーを探します。このフォルダーは通常 Documents and Settings¥<ユーザー名>¥Application Data にあります。Application Data フォルダーは隠しフォルダーなので、Windows エクスプローラでこれらを表示させるよう設定してください。ユーザー名が "Joe" で、Documents and Settings フォルダーが C ドライブにある場合、あなたのプロファイルフォルダーは C:¥Documents and Settings¥Joe¥Application Data¥Phoenix にあります。この Phoenix フォルダーを削除すればプロファイルデータが削除されます。

Mac OS X

自分のホームフォルダーの中にあるライブラリフォルダーを開き、 Phoenix フォルダー (~/Library/Phoenix/) を削除してください。

その他のプラットフォーム

~/.phoenix を削除してください。

既知の問題

ここには、Firefox 0.8 に関する既知の問題の一部が載っています。新たにバグを報告する前にこのリストをお読みになり、今後も定期的にチェックするようにしてください (リリースに新しいバグが見つかった場合は更新されます)。

  • Firefox は Mozilla Application Suite から多くのコードを利用しています。Firefox に含まれる問題の多くは、実際には Mozilla のコアコードにおける問題です。ページのコンテンツや接続性に関する問題を発見した場合、それはおそらく Mozilla の問題ですから、Bugzilla では Firefox プロダクトではなく Browser プロダクトに対して報告してください。
  • Windows 版では、メニューやブックマークツールバー上のフォルダーが正しく折り畳まれないことがあります。
  • テーマインストールの "Use this theme" (このテーマを使用する) チェックボックスは正しく機能しませんので、Tools (ツール) メニューから Options (オプション) を選択し、Themes (テーマ) をクリックして切り替える必要があります。また、一部のテーマは再起動しないと完全に切り替わらないことがあります。
  • 一部のユーザーから、ダウンロードウィンドウのパフォーマンスに問題があるとの報告が寄せられています。ダウンロードウィンドウの表示が遅いと感じられる場合は このパッチ をインストールしてみてください。
  • 複数の拡張機能を同時にインストールすると、前にインストールした拡張機能のインストールダイアログが、その次に拡張機能をインストールするときに表示されてしまうことがあります。このような問題を防ぐため、ひとつの拡張機能をインストールするたびに、一度ブラウザーを再起動することをお勧めします。

Firefox に関するその他の問題、よくある質問、便利な使い方、一般的なヘルプについては、David Tenser の大変便利なヘルプサイト Firefox Help や、MozillaZine によって運営されている Firefox フォーラム をチェックしてください。

トラブルシューティング

前のバージョンの Firefox からアップグレードする際、あらかじめ すべての拡張機能を無効に しておけば、新たにプロファイルを作成する必要はありません。拡張機能を無効にするには、前のバージョンの Firefox を開いて、Tools (ツール) メニューの Options (オプション) を選択し、Extensions (拡張機能) パネルをクリックします。リストからそれぞれの拡張機能をクリックし、Disable Extension (拡張機能を無効にする) を選択してください。すべて無効にしたら OK をクリックして Options ウィンドウを閉じてください。これで安心して新しいバージョンの Firefox をインストールできます。

Firefox をインストールしたら、前にインストールしていた拡張機能が Firefox の新バージョンに対応しているかどうか確認してから、再度インストールしてください。今後この手順は自動化される予定です。

前のバージョンの Firebird や Phoenix で拡張機能を有効にしたまま同じプロファイルを使用して問題が起きた場合は、前のバージョンのブラウザーを起動し、拡張機能を無効にしてから、再度新しいバージョンの Firefox を起動してください。

ブックマーク、ダウンロード、ウィンドウの配置、ツールバー、履歴その他の設定に関して不可解な問題に遭遇した場合は、バグを報告する前に新しいプロファイルを作成して問題が再現するかどうか確かめてください。プロファイルを新規作成するには、コマンドライン引数 -P を付けて Firefox を起動し、Manage Profiles (プロファイルを管理) ボタンをクリックして Create Profile (プロファイルを作成) を選択してください。設定ファイル (ブックマーク、保存したパスワードなど) をひとつずつ手作業で移行し、そのたびに問題が再現するかどうか確認してください。特定のプロファイルデータが問題を起こすことが分かった場合は、バグを登録してそのファイルを添付してください。

よくある質問

  1. どのような協力ができますか?

    私たちは少しでも多くの配布経路を必要としています。できる限り多くの友人や家族、同僚を Firefox に移行させることがあなたの使命です。もしあなたが学生なら、学校で Firefox を配布してください。リリースについて Slashdot [日本語版] やその他のニュースサイトに記事を投稿してください。あなたの blog から情報を流してください。例えばあの Kazaa のように、インターネットを通じた大規模な配布が可能です。Firefox の話題をどんどん広めてください!

  2. 私は別のブラウザーを使っています。なぜなら...

    もしあなたが Firefox を使っていないのなら、開発チームにその理由を教えてください。私たちは Firefox フォーラム に寄せられたフィードバックに目を通しています。

  3. なぜ私が送ったメールに返事をくれないのですか?

    フォーラム をご利用ください。Firefox チームは常時このフォーラムをチェックしています。私たちは毎日多くのメールを受け取っているので、あなたが送ったメールは見過ごされる可能性もあります。

  4. テーマとアドオン (拡張機能) はどこで手に入りますか?

    テーマ と 拡張機能 は Firefox Help からダウンロードできます。

  5. Firefox の作業をしているのは誰ですか?

    現在の開発スタッフは、Ben Goodger (Mozilla Foundation)、Brian Ryner (IBM)、Pierre Chanial、Blake Ross、Dave Hyatt、そして Mozilla のコードベースに貢献している多数のコミュニティのメンバーです。Asa Dotzler、Leaf Nunes と QA コンタクトチームが、バグの選別、リリース、その他諸々の作業を処理しています。詳しくは Help (ヘルプ) メニューの About Mozilla Firefox (Mozilla Firefox について) に書かれているクレジットをご覧ください。

  6. Firefox のソースはどこにありますか?

    cvs.mozilla.org にあります。Mozilla トランクと mozilla/browser と mozilla/toolkit を組み合わせたものになります。

  7. Firefox は、ユーザーインターフェイスにちょっとだけ手を加えた、ただの Mozilla では?

    Firefox には数々の独自機能や数え切れないほどの改良を加えており、まったくの別物です。プロジェクト開始以来、すでに 9 万行以上のコードが CVS ディレクトリーに追加、変更されています。

  8. Phoenix Firebird の名称に何が起きたのですか?

    商標の問題から名称を変更する必要がありました。新名称は Firefox です。詳しくは プレスリリース と よくある質問 をご覧ください。

  9. メールクライアントはどこにあるのですか?

    現在フルセットの Mozilla Application Suite を使っている人の多くは、それをメールやニュースの購読にも利用していると思います。Firefox にはブラウザー機能しかありませんので、そのような使い方はできませんが、Firefox に乗り換えたからといって Mozilla のメールクライアントを使うことをあきらめる必要はありません。Firefox ブラウザーにぴったりな、新しいメールアプリケーション Mozilla Thunderbird をご利用ください。
