Firefox リリースノート

Firebird 0.6.1

Mozilla Firebird (元 Phoenix) は、Galeon、K-Meleon、Camino™ と同じように Mozilla のブラウザーコンポーネントを再設計したものですが、XUL ユーザーインターフェイス言語を使って記述され、クロスプラットフォームであるように設計されています。Mozilla Firebird に関する 詳しい情報 もご覧ください。

このドキュメントには、Mozilla Firebird 0.6.1 のリリースに関する 新機能、ダウンロード・インストール方法、既知の問題、よくある質問 が載っています。Bugzilla にバグを報告する前に、このリリースノートと バグ登録方法 をお読みください。

Mozilla Firebird に関するあなたのフィードバックをお待ちしています。MozillaZine [日本語版] が親切にも提供してくれている Mozilla Firebird フォーラム にぜひ参加してください。

新機能

Mozilla Firebird 0.6.1 は、Mozilla Firebird 0.6 で見つかったいくつかの重大な問題を修正した バグ修正マイナーリリース です。その他にも、次のような細かいバグの修正や変更をいくつか行いました。

  • フォームのオートコンプリート (自動補完) でクラッシュするバグの修正
  • 多くのブックマークレットが使えなくなった、DOM のセキュリティ制限に関するバグの修正
  • リッチテキスト編集 サポート
  • 切り取り、コピー、貼り付けのためのツールバーボタン
  • ツールバーアイコンのアップデートをはじめとする、洗練されたデフォルトテーマ
  • 新しいアプリケーションアイコン

Mozilla Firebird 0.6 の新機能

  • 新しいデフォルトテーマ

    Mozilla Firebird 0.6 は、Arvid Axelsson が作成した素晴らしい Qute テーマをベースとした、さっぱりとした斬新で魅力的なテーマが人目を惹いています。また、新しいテーマには Bookmark Manager ツールバー用のアイコンが付いています。

  • 再設計された設定ウィンドウ

    新しい Options ウィンドウでは、もっともよく使われる設定が 7 つの使いやすいパネルにまとめられており、それぞれに説明的なアイコンセレクタが付いています。

  • プライバシーオプションの改良

    フォームデータ、履歴、キャッシュ、Cookie など、すべての個人情報を、ワンクリック (と対話ダイアログ) で消去できるようになりました。

  • ブックマークの改良

    ブックマークに関する機能向上はいくつもありますが、その中でも、Bookmarks メニューにコンテキストメニューが付きました。メニューの中のブックマーク上で右クリックしてみてください。

  • Talkback が有効に

    Mozilla Firebird がクラッシュしたときに Talkback 情報を送れるようになりました。これは、開発者がより簡単に深刻なバグを修正できるようにするものです。より詳しい情報は FAQ の Talkback について をお読みください。

  • 自動画像リサイズ機能

    この機能を使うと、Mozilla Firebird は、ウィンドウサイズより大きい画像を全体が見られるように縮小表示します。このとき、画像上のカーソルが変わって、クリックすると画像が元のフルサイズに戻ることを知らせてくれます。この機能を無効にする 方法もあります。

  • スムーズスクロール

    この機能を無効にする 方法もあります。

  • より多くの設定項目にアクセス

    アドレスバーに about:config と入力し Enter キーを押すことで、Options ウィンドウには現れていない、多くの上級者向け設定が編集可能になりました。

  • プロファイルクローム

    Mozilla Firebird の拡張機能を、現在のプログラムフォルダーの代わりに、ユーザー自身のプロファイルフォルダーにインストールできるようになりました。これによって、拡張機能をすべて手放すことなく、新しいバージョンの Mozilla Firebird にアップグレードすることが可能になります。なお、これについては、拡張機能の作者がこの仕組みを利用できるように自作アプリケーションをアップデートしていることが条件です。一部の拡張機能は、まだオプションとしてこの仕組みを提供していません。

  • Mac OS X サポート

    Mac OS X 版の Mozilla Firebird が利用可能になりました。完成度はまだいまいちですが、これはスタートです。

  • その他数多くのバグの修正

    Phoenix 0.5 以降、すべて把握するのが大変なくらい、数多くのバグの修正がおこなわれました。0.6 はより良いものになっています。私たちを信じてください。

ダウンロード・インストール・アンインストール

お読みください : Mozilla Firebird 0.6.1 用に新しいプロファイルを作成する必要があります。 新しいプロファイルを作成するには、MozillaFirebird.exe -p を実行して Mozilla Firebird を起動し、"Create Profile" ボタンをクリックしてください。

また、今あるファイルにただ上書きするのではなく、古い Mozilla Firebird のディレクトリーを削除する必要があります。 そうしないと問題が起こる可能性があります。最初から正しいインストールをおこない、新しいプロファイルでテストしない限り、Mozilla Firebird に関するどんなバグも登録すべきではありません。Mozilla Firebird がさらに安定すれば、この作業は必要なくなると思いますが、それまではこれらの措置が絶対に必要です。

mozilla.org は、Windows、Linux、Mac OS X 向けの Mozilla Firebird バイナリを提供しています。

  • Windows: MozillaFirebird-0.6.1-win32.zip
  • Linux: MozillaFirebird-0.6.1-i686-pc-linux-gnu.tar.gz
  • Mac OS X: MozillaFirebird-0.6.1-mac.dmg.gz
  • 貢献者より提供されたビルド: これらは非公式なビルドで、mozilla.org のビルドとは設定が異なる場合もあります。
  • ローカライズ (地域化) ビルド: MLP のボランティアによって提供されています。
    • 日本語版: Windows フルインストーラ
    • 日本語版: Mac OS X ディスクイメージ

バイナリをダウンロードしたら、Winzip か gzip を使って、Mozilla Firebird を空のディレクトリーに解凍し、MozillaFirebird.exe を実行してください。

Mozilla Firebird のアンインストールは、Mozilla Firebird ディレクトリーを削除するだけです。この作業をしても、あなたのプロファイルは、今後も Mozilla Firebird のインストールによって利用できる場所に保存されています。Mozilla Firebird の プロファイルフォルダー を削除したければ、Windows なら Application Data ディレクトリーにある Phoenix ディレクトリーを、Linux ならホームディレクトリーから .phoenix ディレクトリーを削除してください。

既知の問題

ここには、Mozilla Firebird 0.6.1 に関する既知の問題の一部が載っています。新たにバグを報告する前にこのリストをお読みになり、今後も定期的にチェックするようにしてください (リリースに新しいバグが見つかった場合は更新されます)。

  • Mozilla Firebird は Mozilla Application Suite から多くのコードを利用しています。Mozilla Firebird に含まれる問題の多くは、実際には Mozilla のコアコードにおける問題です。ページのコンテンツや接続性に関する問題を発見した場合、それはおそらく Mozilla の問題ですから、Bugzilla では Mozilla Firebird プロダクトではなく Browser プロダクトに対して報告してください。
  • Windows 版では、メニューやブックマークツールバー上のフォルダーが正しく折り畳まれないことがあります。
  • Profile Manager の "Don't ask startup" チェックボックスは機能しません。
  • テーマインストールの "Use this theme" チェックボックスは正しく機能しませんので、Tools > Options > Themes から切り替える必要があります。また、一部のテーマは再起動しないと完全に切り替わらないことがあります。
  • フォームのオートコンプリート (自動補完) の無効化は機能しません。
  • Mac OS X 版の Mozilla Firebird は新しく、まだ完成度が十分ではありません。プラットフォームと一致しないところがあると思います。

Mozilla Firebird に関するその他の問題、よくある質問、便利な使い方、一般的なヘルプについては、David Tenser の大変便利なヘルプサイト Mozilla Firebird [日本語版] や、MozillaZine [日本語版] によって運営されている Mozilla Firebird フォーラム をチェックしてください。

よくある質問

  1. どのような協力ができますか?

    私たちは少しでも多くの配布経路を必要としています。5 人の友人や家族、同僚を Mozilla Firebird に移行させることがあなたの使命です。もしあなたが学生なら、学校で Mozilla Firebird を配布してください。リリースについて Slashdot [日本語版] やその他のニュースサイトに記事を投稿してください。あなたの blog から情報を流してください。例えばあの Kazaa のように、インターネットを通じた大規模な配布が可能です。Firebird の話題をどんどん広めてください!

  2. 私は別のブラウザーを使っています。なぜなら...

    もしあなたが Mozilla Firebird を使っていないのなら、開発チームにその理由を教えてください。私たちは Mozilla Firebird フォーラム に寄せられたフィードバックに目を通しています。

  3. なぜ私が送ったメールに返事をくれないのですか?

    フォーラム をご利用ください。Firebird チームは、フォーラムに寄せられたフィードバックを欠かさず読んでいます。メールを送ってもおそらく無視されます。

  4. テーマとアドオン (拡張機能) はどこで手に入りますか?

    テーマ と 拡張機能 をご利用ください。

  5. Mozilla Firebird の作業をしているのは誰ですか?

    Dave Hyatt、Blake Ross、Pierre Chanial、noririty、Ben Goodger、Dean Tessman、Mike Kaply、Jan Varga、Brian Ryner が開発に参加しています。Asa Dotzler は、リリース、バグの選別、Weウェブサイトの他雑用を手伝ってくれます。David Tenser はドキュメントや Weウェブサイトリースに関して協力してくれます。Arvid Axelsson はデフォルトテーマをメンテナンスしています。Mozilla Firebird に関する評価、協力の申し出、その他一般のメールは フォーラム に送ってください。

  6. Mozilla Firebird のソースはどこにありますか?

    cvs.mozilla.org にあります。Mozilla トランクと mozilla/browser と mozilla/toolkit を組み合わせたものになります。

  7. Mozilla Firebird は、ユーザーインターフェイスにちょっとだけ手を加えた、ただの Mozilla では?

    すでに 4 万行以上のコードが Mozilla のコードに追加・変更されていますので、別物として捉えるべきです。

  8. Phoenix の名前に何が起きたのですか?

    商標の問題から名称を変更する必要がありました。新名称は Mozilla Firebird です。

  9. メールクライアントはどこにあるのですか?

    現在フルセットの Mozilla アプリケーションスイートを使っている人の多くは、それをメールやニュースの購読にも利用していると思います。Mozilla Firebird にはブラウザー機能しかありませんので、そのような利用はできませんが、Mozilla Firebird への乗り換えたからといって Mozilla のメールクライアントの利用をあきらめる必要はありません。Mozilla Firebird ブラウザーにぴったりな、新しいメールアプリケーション Mozilla Thunderbird をご利用ください。
