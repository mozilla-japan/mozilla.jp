Android 版 Firefox リリースノート

バージョン 4.0.1 — 2011/04/28 リリース

Firefox を好きなところへ持ち出そう。Android 版 Firefox はデスクトップ版 Firefox と同じ技術プラットフォームを基盤としながら、モバイル端末でのブラウジングに最適化されています。このリリースノートでは、新機能や改良点、既知の問題、よくある質問を掲載しています。いつものように、フィードバックを提供 するか バグを報告 して、製品の改良にご協力ください。サポートが必要なら、Firefox for Mobile Support Forum をご利用ください。

Firefox 4.0.1 での変更点

このマイナーアップデートでは、目に見える機能は一切追加していませんが、いくつかの重要なバグを修正し、安定性、パフォーマンス、ローカライズ版における整合性の向上を行いました。また、これまで特定の端末でクラッシュにつながっていたと思われる問題を修正しました。

  • 様々なクラッシュ問題の修正 (Bug 643307、Bug 644012、Bug 626051 - Cyanogen Mod)
  • Bug 644735 - ローカライズの修正 - 一部文字列が長すぎて UI に収まりきらなかった問題
  • Bug 643836 - よりスムーズなパンと格子模様表示の削減 - ディスプレイポートの更新
  • Bug 644613 - バーチャルキーボードに関する問題
  • Bug 643038 - about:firstrun ページのレイアウト修正
  • Bug 649115 - Spark キャンペーン - 5 月末までスタートページにリンクを設置

本バージョンでは、ユーザーの目に見える機能は追加されていません。

開発チームでは、リリースのたびに、パフォーマンスや端末互換性の改善、一方でメモリーと CPU 使用率の削減に継続的に取り組んでいます。実装されている機能の一覧は 機能と特長 をご覧ください。

アドオン

  • Android 版 Firefox 向けの 拡張機能 は 150 種類以上あり、その数は増え続けています。そのうち数十個はすでに最新版へ対応しています。素晴らしいコミュニティのおかげで、Firefox 4 対応のエキサイティングな拡張機能の数は絶えず増え続けています。Mobile Add-on Cup の優秀作品をはじめとする素敵なモバイル向け拡張機能をぜひお試しください。

開発者向けの新機能と改良点

  • 新しい素敵なアドオンを作ったり、既成のモバイル用アドオンをリリース候補版へ対応させるなら、今がその時期です。インタフェースはすべて安定しており、必要なツールと資料はすべて 開発者センター に揃っています。
  • IPv6 に対応しました。
  • CSS プロパティの「overflow: scroll」「overflow: auto」に対応しました。
  • Beta 4 で 再起動不要なアドオン に対応しました。
  • 同じく Beta 4 で HTML5 のオンライン・オフラインイベント に対応しました。

その他、ベータ版で追加もしくは改良された機能

  • 画面に合わせて文字サイズを変更する機能を追加し、文章を読みやすくしました。ダブルタップで拡大します。この機能は Firefox の設定画面で有効化できます。
  • ユーザービリティ：Android のハードウェアとソフトウェアキーボードに対応し、Android 形式のメニュー、ロケーションバーでのコピー＆ペースト、スマートスクリーン上で文字入力中の検索エンジン表示、アドオンマネージャーでのおすすめアドオン参照など、様々な機能を追加しました。
  • ファイル処理：ファイルのアップロードに対応するとともに、サイトメニューに「PDF として保存」を追加しました。
  • ローカライズ：アラビア語とペルシア語の (各言語のフォントがインストールされている端末で) テキストレンダリングを改善しました。
  • Sync：セットアップを簡素化する新 UI を採用し、独自サーバーに対応しました。
  • 動画：オープン動画の全画面表示に対応し、音声のパフォーマンスを向上させました。
  • 開発者向け機能：localStorage API、端末の傾き検知 (加速度センサー連携) に対応しました。
  • Sync：Firefox Sync が標準装備となり、Firefox の履歴、ブックマーク、開いているタブ、フォーム入力履歴、パスワードを、デスクトップとモバイル間で同期できるようになりました。
  • アドオン：個人の好みに合わせて Firefox の機能や見た目を変更できるアドオン (小さな追加機能) に対応しました。
  • ピンチもしくはダブルタップによるページの拡大と縮小が可能になりました。
  • 目的のサイトを簡単に確認して開けるよう、タブをサムネイルとして開くようにしました。
  • 現在地に関連性の高い情報を取得できる 位置情報通知機能 に対応しました。
  • サイトメニュー内の「PDF として保存」で、航空券や道順などの重要な情報をオフラインで見られるようアーカイブできます。
  • サイトメニュー内の「ページ中を検索」で、ウェブページ上の文字列を素早く検索できます。
  • サイトメニュー内の「ページを共有」で、メールや Facebook、Twitter、Google リーダーへコンテンツを送信できます。
  • サイトメニュー内の「パスワードを消去」で、信用できなくなったサイトのログイン情報を消去できます。
  • サイトメニュー内の「検索エンジンを追加」で、新しい検索エンジンをスマートスクリーンへ素早く追加できます。
  • リンクを長押しすると表示されるコンテキストメニューで、リンクを新しいタブで開いたり共有したりできます。
  • スマートタップ機能により、リンクやウィジェット、その他ウェブコンテンツを正確にタップできます。
  • パスワードマネージャーでサイトのログイン情報を記憶させれば、以後文字入力せずにログインできます。

開発者向け情報

Firefox は、高速、パワフル、かつ美しいモバイル向けウェブアプリケーションやアドオンの作成に適した、使いやすくオープンな開発プラットフォームです。HTML5、CSS、JavaScript など最新のウェブ技術を使って、より素早く簡単に革新的な機能を実装し、リーチを広げることが可能です。スマートフォンが持つデバイス固有の機能を活用して素晴らしいサイトやアプリを開発したり、ウェブの使い方を変えるアドオンを作成したりしましょう。Mobile Developer Network を訪れて、膨大な数の端末へリーチできる革新的なサイトを構築する方法を学んでください。

モバイル向けアドオンの開発

デスクトップ版 Firefox と Android 版 Firefox はいずれも同じプラットフォームを基盤としていますが、両者には若干の違いがあるため、アドオン作者は Android へ対応させるため少し手を加える必要があります。新機能として、再起動不要なアドオン を作れるようになりました。詳しくは、MozillaWiki にある Electrolysis と Add-ons Training Week のドキュメントを参照してください。

バックエンドの特長

  • デスクトップ版 Firefox 4 と同じブラウザーエンジン
  • Mozilla 独自の Jägermonkey (イェガーモンキー) JIT コンパイラを統合した高速 JavaScript エンジン
  • 優れたウェブサイト互換性
  • ARM プロセッサーに最適化
  • 位置情報通知機能

HTML5 をはじめとするウェブ標準技術

  • ウェブワーカー
  • オフラインストレージ
  • <canvas>
  • <svg>
  • オンライン・オフラインイベント
  • ネイティブ JSON
  • <audio>
  • <video>

既知の問題

  • デスクトップ版 Firefox のプロファイルのサイズが大きい状態で Firefox Sync を使うと、初回同期の際にパフォーマンスの問題が生じる場合があります。今後のバージョンで改善に取り組みます。
  • 自作のアドオンをまだ Firefox 4 へ対応させていない開発者は、MozillaWiki にある Electrolysis と Add-ons Training Week のドキュメントを参照して、Firefox 4 対応に役立ててください。
  • Adobe Flash などのプラグインには対応していません。現時点で Flash プラグインを有効にすると、アプリケーション全般のユーザー体験を損ねるためです。
  • Google ドキュメントあるいは Etherpad の文書やスプレッドシート内のコンテンツを編集することができません。

その他、既知のバグ

  • Bug 617298 - SwiftKey を使うと、クラッシュしたり、入力欄に文字を入力できないなど、いくつかの問題があります。
  • Bug 602252 - 一部の Motorola 製端末、特に Motorola Droid (別名 Milestone) で一部のページを Firefox で読み込むと、長時間フリーズする場合があります。
  • Bug 601222 - Android 2.1 が動作している Samsung Galaxy S では Firefox の JIT コンパイラが無効になります。このため、他の端末よりも Firefox の JavaScript 実行速度が低下します。
  • Bug 624652 - 特定の状況で、Nexus S 端末で Firefox を実行中に Android 2.3.1 がフリーズする場合があります。これは Android 2.3.3 へシステムを更新することで改善するようです。
  • Bug 622839 - HTML5 動画の読み込みが遅くなり、再生中画面上にノイズが見られる場合があります。
  • Bug 628269 - 再読み込みの際にページが拡大表示されてしまいます。

システム要件

Firefox 4 は Android 2.0 以上に対応しています。

Firefox 4 は 512 MB の RAM と約 17 MB のストレージを必要とします。ただし、Android 2.2 では、Android に付属している Settings アプリを使用して、Firefox 本体とユーザーデータを SD カードへ移すことができます。

Firefox 4 は ARMv7 プロセッサー搭載の端末を必要とします。インストール前に 対応機種 をご確認ください。

ダウンロード

Firefox 4 をダウンロードするには、Android 端末で firefox.com/m を開いてください。お使いの端末がシステム要件を満たしているかどうか、インストール前にあらかじめ 対応機種 をご確認ください。

開発者の皆さんは、パソコンに Android 版 Firefox のエミュレータをインストールして、テスト、フィードバックの提供、アドオンの開発を行えます。パソコン用のエミュレータは開発目的に限って公開しているもので、完全な機能は備えていません。
